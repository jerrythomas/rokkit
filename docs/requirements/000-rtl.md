# RTL Support Requirements

> Cross-cutting requirements for right-to-left language support.

## 1. Overview

RTL support should derive from HTML `lang` attribute automatically. This is a cross-cutting concern that affects `@rokkit/core`, `@rokkit/states`, `@rokkit/actions`, and all UI components.

## 2. Implicit RTL Detection

```javascript
// @rokkit/states/vibe.svelte.js
import { detectRTL } from '@rokkit/core'

export const vibe = $state({
  style: 'rokkit',      // Theme style
  palette: 'default',   // Color palette
  mode: 'light',        // 'light' | 'dark'
  direction: detectRTL() // 'ltr' | 'rtl' - derived from document.documentElement.lang
})
```

## 3. Implementation

- Auto-detect from `<html lang="ar">` or similar RTL languages
- Expose as reactive store value on `vibe.direction`
- Components read direction from vibe store
- Arrow key navigation adapts automatically (Left/Right swap in RTL)
- `@rokkit/core` provides `detectDirection()` and `isRTL()` utilities (already exist)

## 4. Affected Packages

| Package | What Changes |
|---------|-------------|
| `@rokkit/core` | `detectDirection()`, `isRTL()` — already implemented |
| `@rokkit/states` | `Vibe` — add `direction` property (already has it) |
| `@rokkit/actions` | `navigator` — swap Left/Right in RTL mode |
| `@rokkit/ui` | All components — respect direction for layout |
| `@rokkit/themes` | CSS — `[dir="rtl"]` or logical properties |

## 5. Vibe Store Naming

Consider renaming for clarity:
- `vibe` → `theme` or `skin` (more familiar terminology)
- Keep `vibe` as alias for backwards compatibility

## 6. Current Status

- `detectDirection()` and `isRTL()` exist in `@rokkit/core/utils.js`
- `Vibe` class in `@rokkit/states` has a `direction` property
- No UI components currently read or adapt to direction
- No CSS logical properties in use yet
