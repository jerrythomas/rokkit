# Rokkit — LLM Reference Index

> Data-driven, accessible UI components for Svelte 5

## Quick Setup

**Always use the CLI — manual setup leads to configuration errors.**

1. Create a SvelteKit project:
   ```bash
   npm create svelte@latest myapp
   cd myapp
   npm install
   ```

2. Initialize Rokkit:
   ```bash
   npx @rokkit/cli@latest init
   ```

   The `init` command is interactive. It prompts for:
   - **Color palette:** `default` (orange/pink/sky) | `vibrant` (blue/purple/sky) | `seaweed` (sky/green/blue) | `zen-sumi` (OKLCH ink-on-paper) | `custom`
   - If `custom`: primary, secondary, accent, surface colors (Tailwind palette names, e.g. `orange`)
   - **Icon collection:** Rokkit built-in | Rokkit + custom collection
   - If custom: path to custom icon collection JSON
   - **Theme styles** (multi-select, min 1): `rokkit` | `minimal` | `material` | `frosted` | `zen-sumi`
   - **Theme switching:** `system` (prefers-color-scheme) | `manual` (light/dark toggle) | `full` (light/dark + style variants)

   Files written: `rokkit.config.js`, `uno.config.js`, `src/app.css`, `src/app.html`

   See [cli.txt](/llms/cli.txt) for full CLI reference (includes `rokkit skills` for installing AI skill guides).

## Verify & Fix

After setup, or when things seem broken:

```bash
npx @rokkit/cli@latest doctor
```

Runs 4 checks:
- `rokkit.config.js` exists
- `uno.config.js` uses `presetRokkit()`
- `src/app.css` has theme imports
- `src/app.html` has theme init script

Auto-fix 3 of 4 checks:
```bash
npx @rokkit/cli@latest doctor --fix
```

Exit codes: `0` = all pass, `1` = any failures.

## Troubleshooting

### `rokkit.config.js` missing

`doctor --fix` can auto-generate an empty config. If running manually:
```js
// rokkit.config.js — minimal valid config
export default {}
```

### `app.css` missing theme imports

Prepend to `src/app.css`:
```css
@import '@unocss/reset/tailwind.css';
@import '@rokkit/themes/base.css';
@import '@rokkit/themes/rokkit.css';  /* also minimal, material if needed */
```

### `uno.config.js` does not use `presetRokkit()` (not auto-fixable)

`doctor --fix` cannot overwrite an existing `uno.config.js`. Add the preset manually:

```js
// uno.config.js
import { defineConfig } from 'unocss'
import { presetRokkit } from '@rokkit/unocss'

export default defineConfig({
  presets: [presetRokkit()]
})
```

### `src/app.html` missing theme init script

A small pre-paint script reads the persisted theme from `localStorage` and applies
`data-style` / `data-mode` / `data-density` / `data-skin` before first paint, preventing a
flash-of-unstyled-content (FOUC). It resolves a `system`/`auto` mode against `prefers-color-scheme`.

**Don't hand-roll it.** `doctor --fix` injects the canonical script (generated by
`@rokkit/unocss`'s `themeInitScript`) using the `storageKey` from your `rokkit.config.js`.

For SSR apps, prefer the `themeHook` in `src/hooks.server.js` — the same script injected
server-side, so there's no flash even on first load:

```js
// src/hooks.server.js
import { themeHook } from '@rokkit/unocss/hooks'
export const handle = themeHook({ storageKey: '<your storageKey>', defaultStyle: 'rokkit' })
```

`themeHook` / `themeInitScript` default the mode to `system` (resolving `prefers-color-scheme`),
so a `system` switcher needs no special-casing.

### Theme init script missing after --fix

Verify that `<body>` is on its own line in `src/app.html`. The script is injected immediately after `<body>`.

## Standard API Pattern

Most selection components share this pattern:

```svelte
<Component
  items={data}                 <!-- the data array -->
  bind:value                   <!-- selected value (bindable) -->
  fields={{ label: 'name' }}   <!-- map data fields to component expectations -->
  onchange={(v, item) => {}}   <!-- selection callback -->
/>
```

## Field Mapping System

Map any data structure without transforming data:

```svelte
<script>
  const users = [
    { userId: 1, fullName: 'Alice', avatarUrl: '/alice.jpg' }
  ]
  const fields = {
    value: 'userId',
    label: 'fullName',
    icon: 'avatarUrl'
  }
</script>

<Select items={users} {fields} bind:value />
```

### Field Roles Reference

Field roles are the keys you use in the `fields` prop. They map from Rokkit's semantic names to your data's property names.

| Field role | Default data key | Description | Used by |
|------------|-----------------|-------------|---------|
| `label` | `'label'` | Display text | All components |
| `value` | `'value'` | Selection value (what `bind:value` receives) | Select, List, Toggle, etc. |
| `icon` | `'icon'` | Icon CSS class | List, Select, Menu, Tree |
| `avatar` | `'image'` | Avatar image URL | List, Select |
| `subtext` | `'description'` | Secondary text below label | List, Select |
| `tooltip` | `'title'` | Tooltip text | All |
| `badge` | `'badge'` | Badge value | List |
| `shortcut` | `'shortcut'` | Keyboard shortcut label | Menu |
| `children` | `'children'` | Nested items for groups/trees | List (groups), Tree |
| `snippet` | `'snippet'` | Named snippet to route the item to | List, Select, Tree |
| `href` | `'href'` | Link URL — renders item as `<a>` | List, Menu |
| `hrefTarget` | `'target'` | Link target (`_blank`, etc.) | List, Menu |
| `disabled` | `'disabled'` | Disables the item | All |
| `expanded` | `'expanded'` | Initial expand state | List, Tree |
| `selected` | `'selected'` | Initial selected state | List |
| `id` | `'id'` | Unique identifier | All |

**Default mapping:** if you pass no `fields` prop, Rokkit uses field names directly — a property named `label` becomes the display text, `value` becomes the selection value, etc.

**Primitive arrays:** strings and numbers work without any field mapping. `['Alice', 'Bob']` gives you both `label` and `value` set to the primitive itself.

```svelte
<!-- Primitives work without fields -->
<Select items={['Alice', 'Bob', 'Carol']} bind:value />

<!-- Object array with explicit mapping -->
<Select items={users} fields={{ label: 'fullName', value: 'userId' }} bind:value />
```

**Same pattern everywhere:** `items` + `fields` + `bind:value` works identically across List, Select, MultiSelect, Tree, Tabs, Toggle, Table, Menu — no adapters needed.

## Theming

### Data Attributes

The theme is driven by data-attributes on `<html>`/`<body>`:

```html
<body
  data-style="rokkit"         <!-- visual personality: rokkit | minimal | material | frosted | zen-sumi -->
  data-mode="dark"            <!-- light | dark -->
  data-density="comfortable"  <!-- compact | comfortable | cozy -->
>
```

### Applying & persisting the theme (don't hand-roll this)

Do NOT write `data-mode` by hand, read/write `localStorage`, or build your own save
logic. Use the **`vibe` store** (`@rokkit/states`) + the **`themable` action**
(`@rokkit/actions`) + **`ThemeSwitcherToggle`** (`@rokkit/app`). The action loads from
storage on mount, writes `data-style`/`data-mode`/`data-density` (mirrored to `<html>`
to match the flash-prevention init script), saves on every change, and syncs across tabs
via the `storage` event.

```svelte
<!-- once, at the app root (e.g. +layout.svelte) -->
<script>
  import { vibe } from '@rokkit/states'
  import { themable } from '@rokkit/actions'
</script>
<svelte:body use:themable={{ theme: vibe, storageKey: 'app-theme' }} />
```

```svelte
<!-- anywhere: the mode control. Drives vibe.mode → themable writes data-mode -->
<script>import { ThemeSwitcherToggle } from '@rokkit/app'</script>
<ThemeSwitcherToggle />                    <!-- default variant="triad": system | light | dark group -->
<ThemeSwitcherToggle variant="single" />   <!-- cycling light/dark-only button -->
```

`vibe` is a singleton with reactive `mode` (`light`|`dark`), `style`
(`rokkit`|`minimal`|`material`), `density`, `direction`, plus `load(key)` / `save(key)` /
`update(partial)`. Switch the visual style by setting `vibe.style = 'minimal'` — no manual
DOM writes. (`storageKey` must match the key used by the `rokkit init` flash-prevention
script in `app.html`.)

### UnoCSS Utility Classes (how to apply tokens)

Do NOT use CSS variables in `<style>` blocks — they are not theme-aware. Use utility classes:

```svelte
<!-- ✅ Correct: theme-aware, responds to data-mode switching -->
<div class="bg-paper-soft border border-paper-edge text-ink rounded-lg p-4">
  <h2 class="text-primary font-semibold">Title</h2>
  <p class="text-ink-mute text-sm">Subtitle</p>
</div>

<!-- ❌ Wrong: not theme-aware -->
<div style="background: var(--paper-soft)">...</div>
```

### Named Token Vocabulary (primary authoring model)

Components use **named tokens** that flip automatically under `[data-mode="dark"]`.
The core vocabulary (`tokens: 'core'`):

| Group   | Tokens                                                        | Use                                   |
|---------|---------------------------------------------------------------|---------------------------------------|
| Surface | `paper` · `paper-soft` · `paper-mute` · `paper-edge`          | canvas / card / panel / hairline      |
| Text    | `ink` · `ink-mute` · `ink-soft` · `ink-faint`                 | body / secondary / placeholder / disabled |
| Primary | `primary` · `on-primary`                                      | CTA fill / text on primary            |
| Accent  | `accent` · `accent-soft`                                      | accent fill / tinted accent bg        |
| Status  | `success`/`-soft`, `warning`/`-soft`, `danger`/`-soft`, `error`/`-soft`, `info`/`-soft` | solid / tinted-bg callout |
| Misc    | `focus-ring` · `shadow-tint`                                  | focus ring / shadow color             |

Prefixes: `bg-`, `text-`, `border-` (+ `-t/-b/-l/-r`), `fill-`, `stroke-`, `ring-`,
`outline-`, `divide-`. Example: `bg-paper`, `text-ink-mute`, `border-paper-edge`,
`text-on-primary`, `bg-success-soft text-success`.

Opt into the full 11-shade palette ladder per role with `tokens: 'extended'` (for charts / data-viz).

### Common Patterns

```svelte
<!-- Page background -->
<div class="bg-paper text-ink">

<!-- Card -->
<div class="bg-paper-soft border border-paper-edge rounded-lg p-4">

<!-- Section header -->
<h2 class="text-xs font-semibold text-ink-soft uppercase tracking-wider">

<!-- Primary link -->
<a class="text-primary hover:text-accent transition-colors">

<!-- Input field -->
<input class="bg-paper-mute border border-paper-edge text-ink focus:border-primary focus:outline-none rounded px-3 py-2">

<!-- Status indicators -->
<span class="text-warning">Stale</span>
<span class="text-success">Active</span>
<span class="text-error">Failed</span>

<!-- Table row separator -->
<tr class="border-b border-paper-edge last:border-b-0">
  <td class="px-4 py-2.5 text-ink-mute">...</td>
</tr>
```

### Inline styles: only for dynamic computed values

```svelte
<!-- ✅ OK: computed in JS, can't be a static class -->
<div style="background: {color}22; color: {color}">

<!-- ❌ Not OK: use utility class instead -->
<div style="background: var(--paper-soft)">
```
