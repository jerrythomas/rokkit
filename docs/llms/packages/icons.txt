# @rokkit/icons

> Curated Iconify-compatible icon collections for Rokkit applications. Static JSON bundles — no runtime JavaScript required.

Icons are distributed as [Iconify](https://iconify.design/) JSON files. They integrate with UnoCSS (via `@rokkit/unocss`), the Iconify runtime, or any toolchain that accepts Iconify collections.

## Install

```bash
npm install @rokkit/icons
```

---

## Collections

| Subpath import | Collection prefix | Contents |
|----------------|-------------------|----------|
| `@rokkit/icons` or `@rokkit/icons/ui.json` | `rokkit` | Base UI icons — navigation, actions, states |
| `@rokkit/icons/semantic.json` | `semantic` | Semantic icons with style variants (default for shortcuts) |
| `@rokkit/icons/glyph.json` | `glyph` | Large icon set (600+ icons) for tooling, palettes, docs |
| `@rokkit/icons/light.json` | `light` | Outlined lightweight icons |
| `@rokkit/icons/solid.json` | `solid` | Solid filled icons |
| `@rokkit/icons/twotone.json` | `twotone` | Two-tone icons |
| `@rokkit/icons/components.json` | `components` | Component-type icons (alias for glyph) |
| `@rokkit/icons/auth.json` | `auth` | Auth provider logos (Google, GitHub, Firebase, etc.) |
| `@rokkit/icons/app.json` | `app` | Application-level icons |
| `@rokkit/icons/utils` | — | Utility functions for icon conversion |

The **semantic** collection is the default for component icon shortcuts. Each icon has four variants:
- Default (duotone) — `accordion-closed`
- Solid — `accordion-closed-solid`
- Outline — `accordion-closed-outline`
- Duotone outline — `accordion-closed-duotone-outline`

Set `icons.style` in `rokkit.config.js` to switch all component icons globally (e.g., `style: 'solid'`).

## Usage with @rokkit/unocss (recommended)

`presetRokkit()` registers selected collections automatically. Icons become UnoCSS utility classes:

```html
<!-- Syntax: i-{prefix}:{icon-name} -->
<span class="i-rokkit-ui:menu"></span>
<span class="i-rokkit-ui:action-add"></span>
<span class="i-rokkit-solid:action-check"></span>
<span class="i-rokkit-auth:google"></span>
```

Configure which collections to load in `uno.config.js`:

```javascript
import { presetRokkit } from '@rokkit/unocss'

export default {
  presets: [
    presetRokkit({
      icons: ['ui', 'solid', 'auth']  // load specific collections
    })
  ]
}
```

## Usage with Iconify runtime

Import JSON and register with `@iconify/svelte` or `@iconify/icon-offline`:

```javascript
import { addCollection } from '@iconify/svelte'
import uiIcons from '@rokkit/icons/ui.json'
import authIcons from '@rokkit/icons/auth.json'

addCollection(uiIcons)
addCollection(authIcons)
```

---

## JSON bundle format

Each file is a standard Iconify JSON collection:

```json
{
  "prefix": "rokkit-ui",
  "lastModified": 1234567890,
  "icons": {
    "menu": { "body": "<path d=\"...\" />" },
    "action-add": { "body": "<path d=\"...\" />" }
  }
}
```

## Auth collection (`rokkit-auth`) — providers

Includes logos for: Firebase, GitHub, Google, Microsoft, AWS Cognito, Supabase, Auth0, Clerk, Okta, Apple, Twitter/X, LinkedIn, Facebook, Discord, Slack, and others. Each provider has variants: default, black, white, wordmark, duotone, outline (availability varies by provider).

```html
<span class="i-rokkit-auth:google"></span>
<span class="i-rokkit-auth:github-white"></span>
<span class="i-rokkit-auth:firebase-duotone"></span>
```
