# @rokkit/ui

> Main UI component library for Rokkit. 40+ components covering selection, navigation, forms, feedback, layout, charts, and theming.

See [index.txt](/llms/index.txt) for the full component catalogue, import reference, and standard API pattern.

Individual component references live in [components/](/llms/components/).

## Install

```bash
npm install @rokkit/ui
```

## LockMode

Renders a subtree pinned to a fixed color mode regardless of the document mode. Statically
renders `data-mode` for a correct SSR first paint, then applies the `lockMode` action from
`@rokkit/actions` on hydration to mirror the root's `data-style`/`data-skin`/`data-density`
and observe runtime changes.

```svelte
<script>
  import { LockMode } from '@rokkit/ui'
</script>

<!-- Always renders in dark mode, even inside a light-mode document -->
<LockMode mode="dark">
  <Card>This card is always dark</Card>
</LockMode>

<!-- Light island inside a dark page -->
<LockMode mode="light">
  <Button>Always light</Button>
</LockMode>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `mode` | `'dark'\|'light'` | required | Color mode to lock this region to |
| `children` | `Snippet` | required | Content rendered inside the locked region |
| `class` | `string` | `''` | Additional CSS class on the wrapper `<div>` |

### How it works

- Renders a `<div data-mode={mode}>` so `@rokkit/themes` component CSS (which keys off
  `[data-mode][data-style]`) engages from the first server render.
- Applies `use:lockMode={mode}` (from `@rokkit/actions`) on hydration: mirrors `data-style`,
  `data-skin`, and `data-density` from `document.documentElement` and observes root
  mutations, keeping the locked region's style/skin/density in sync.
- The locked mode is fixed at mount. Re-mount with a different `mode` prop to change it.

Prerequisite: the `@rokkit/unocss` preset must be at version ≥ 1.2.1, which emits light
token vars under `:root, [data-mode="light"]` (making a nested `data-mode="light"` element
re-assert them). Dark works with any version since `[data-mode="dark"]` was always a
standalone selector.

## CommandPalette

Modal command search UI backed by the `commands` registry from `@rokkit/states`.

```svelte
<script>
  import { CommandPalette } from '@rokkit/ui'
  import { commands } from '@rokkit/states'
  import { shortcuts } from '@rokkit/actions'
  import { onMount } from 'svelte'

  let paletteOpen = $state(false)

  onMount(() => {
    return commands.register({
      id: 'palette.open',
      label: 'Open command palette',
      shortcut: 'mod+k',
      global: true,
      run: () => { paletteOpen = true }
    })
  })
</script>

<div use:shortcuts={commands}>
  {@render children()}
  <CommandPalette bind:open={paletteOpen} />
</div>
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `false` | Controls visibility — use `bind:open` |
| `placeholder` | `string` | `messages.command.placeholder` | Input placeholder; falls back to `'Run a command…'` |

### How it works

- Reads `commands.all` from `@rokkit/states` — no separate `items` prop
- Filters by case-insensitive substring match on `label`, `id`, and `keywords`
- Closes itself on Enter (runs command), Escape, and click-outside
- Chrome strings come from `messages.command` (`placeholder`, `noResults`, `label`)

### Data attributes

| Attribute | Description |
|-----------|-------------|
| `data-command-backdrop` | Fullscreen overlay |
| `data-command-palette` | Dialog container (`role="dialog" aria-modal="true"`) |
| `data-command-input` | Filter `<input>` |
| `data-command-list` | Results `<ul>` (`role="listbox"`) |
| `data-command-item` | Result `<li>` (`role="option"`) |
| `[data-active]` | On active `[data-command-item]` |
| `data-command-label` | Label `<span>` inside item |
| `data-command-shortcut` | Shortcut `<span>` inside item (rendered only when `cmd.shortcut` is set) |
| `data-command-empty` | Empty-state `<p>` when no results |

Full reference: [components/command-palette.txt](/llms/components/command-palette.txt)
