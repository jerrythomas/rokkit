# @rokkit/unocss

> UnoCSS preset for Rokkit — wires up color tokens, semantic shortcuts, icon integration, and dark mode in a single `presetRokkit()` call.

## Install

```bash
npm install @rokkit/unocss
```

## Quick Start

```js
// uno.config.js (or vite.config.js)
import { defineConfig } from 'unocss'
import { presetRokkit } from '@rokkit/unocss'

export default defineConfig({
  presets: [presetRokkit()]
})
```

For SvelteKit, also add the UnoCSS Vite plugin:

```js
// vite.config.js
import { defineConfig } from 'vite'
import { sveltekit } from '@sveltejs/kit/vite'
import UnoCSS from 'unocss/vite'
import { presetRokkit } from '@rokkit/unocss'

export default defineConfig({
  plugins: [
    UnoCSS({ presets: [presetRokkit()] }),
    sveltekit()
  ]
})
```

## `presetRokkit(options?)` — The Single Export

Returns a complete UnoCSS `Preset` object. All configuration is optional — defaults work out of the box.

```js
import { presetRokkit } from '@rokkit/unocss'

const preset = presetRokkit({
  skin: {
    primary: 'violet',   // override semantic color → Tailwind palette name
    surface: 'zinc'
  },
  skins: {
    vibrant: { primary: 'blue', secondary: 'purple' }
  },
  icons: {
    app: '@rokkit/icons/app.json',
    solar: '@iconify-json/solar/icons.json'
  },
  themes: ['rokkit'],
  switcher: 'manual',   // 'system' | 'manual' | 'full'
  storageKey: 'rokkit-theme'
})
```

### Options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `skin` | `Record<string, string \| { light?: string, dark?: string }>` | Default semantic palette | Map semantic names to palette names. String value = auto z-flip for dark mode. Object `{ light, dark }` = separate named palettes per mode — rokkit generates an explicit `[data-mode="dark"]` var block. `skin` defines the default skin; `skins` defines named alternatives. `colors` is accepted as a backward-compatible alias for `skin`. |
| `skins` | `Record<string, Record<string, string \| { light?, dark? }>>` | `{}` | Named colormaps — same syntax as `skin`. The default skin's tokens go on `:root`; each non-default skin emits a `[data-skin='name']` CSS block (+ `[data-mode='dark'][data-skin='name']` for dual-palette). Merged with the shipped `BUILTIN_SKINS`. |
| `icons` | `Record<string, string>` | `{ app: '@rokkit/icons/app.json' }` | Additional icon collections (JSON files). Merged with the built-in `rokkit` and `semantic` collections. |
| `icons.style` | `string` | `undefined` | Icon style variant suffix (e.g., `'solid'`, `'outline'`, `'duotone-outline'`). Applied to all semantic icon shortcuts. |
| `icons.collection` | `string` | `undefined` | Override the default icon collection prefix. Default is `'semantic'` (`i-semantic:*`). Set to e.g. `'phosphor'` to use `i-phosphor:*`. |
| `icons.overrides` | `Record<string, string>` | `{}` | Per-icon class overrides. Keys are shortcut names, values are full icon classes. |
| `palettes` | `Record<string, Record<string, string>>` | `{}` | Custom color palettes with shade values. Merged into theme colors so names can be used in `skin`. |
| `overrides` | `Record<string, string \| { light?, dark? }>` | `{}` | Replace named-token defaults and/or emit new custom tokens, all in one block. Reserved names (e.g. `paper-edge`) override the named-token value via cascade order; non-reserved names emit as new `--{name}` CSS vars. Values can be palette refs (`'kami.400'`), raw CSS colors, or `{ light, dark }` objects. Color-valued non-reserved entries auto-get `bg-/text-/border-` shortcuts. |
| `typography` | `{ sans?, mono?, heading? }` | All `null` | Font family CSS custom properties. Values are CSS font-family strings. |
| `shape` | `{ radius? }` | `{ radius: null }` | Shape tokens. `radius` accepts `'sharp'`, `'rounded'`, `'pill'`, or a custom `{ sm, md, lg, xl, full }` object. |
| `themes` | `string[]` | `['rokkit']` | Which CSS themes to activate (informational — does not import CSS; use CSS imports for that). |
| `tokens` | `'core'\|'extended'` | `'core'` | Token vocabulary level. `'core'` emits the named-token shortcuts (`bg-paper`, `text-ink`, etc.). `'extended'` also emits the full 11-shade palette ladder per role (for charts / data-viz). |
| `switcher` | `'system' \| 'manual' \| 'full'` | `'manual'` | Theme switcher mode hint (used by app-level UI, not by the preset itself). |
| `storageKey` | `string` | `'rokkit-theme'` | `localStorage` key for persisting theme selection. |

## What `presetRokkit` Configures

### Included Sub-presets

- **`presetWind3`** — Tailwind CSS v3-compatible utility classes
- **`presetIcons`** — CSS icon classes (`i-{collection}:{name}`) via `@iconify/utils`
- **`presetTypography`** — Typography prose utilities

### Color Palette Tokens

In the default `tokens: 'core'` mode the preset emits **named tokens** as complete color values for each semantic role. These use the color space configured in `colorSpace` (default `'rgb'`):

```css
--primary:       rgb(249,115,22);   /* shade 500 */
--primary-soft:  rgb(254,243,199);  /* tinted background */
--paper:         rgb(248,250,252);  /* surface shade 50 */
--ink:           rgb(15,23,42);     /* surface shade 900 (inverted) */
/* ... all named tokens ... */
```

These named tokens are consumed by `@rokkit/themes` CSS and all component utility classes (`bg-primary`, `text-ink`, `bg-paper`, etc.).

When `tokens: 'extended'` is set (globally or per-role), the preset also emits the full 11-shade palette ladder (`--color-{role}-50` through `--color-{role}-950`) as well as z-alias vars. Use this for charts and data-viz that need direct shade access.

### Named Token Shortcuts

For every named token (`paper`, `paper-soft`, `paper-mute`, `paper-edge`, `ink`, `ink-mute`, `ink-soft`, `ink-faint`, `primary`, `accent`, `accent-soft`, `success` / `warning` / `danger` / `error` / `info` (+ `*-soft`), `on-primary`, `focus-ring`), the preset emits utility classes with these prefixes:

| Prefix | CSS property | Notes |
|--------|--------------|-------|
| `bg-` | `background-color` | not emitted for `on-primary`, `focus-ring`, `shadow-tint` |
| `text-` | `color` | |
| `border-` | `border-color` | |
| `border-t-` / `-b-` / `-l-` / `-r-` | per-side `border-color` | |
| `ring-` | `--un-ring-color` | emitted only for `focus-ring` (and `*-ring`-suffixed override tokens) |
| `fill-` | `fill` | for SVG |
| `stroke-` | `stroke` | for SVG |

```html
<!-- Hairline border in the canonical etched-edge tone -->
<div class="border border-paper-edge">…</div>

<!-- Top-only divider -->
<div class="border-t border-t-paper-edge">…</div>

<!-- Focus ring -->
<button class="ring ring-focus-ring">…</button>
```

**Prefer these utility classes over wrapping `var(--paper-edge)` etc. in a custom property.** CSS substitutes `var()` inside a custom property's value at the *declaring* element and inherits the resolved string — so `--hairline: 1px solid var(--paper-edge)` declared on `:root` won't flip when `--paper-edge` changes under `[data-mode="dark"]`. Utility classes resolve `var()` at the consuming element, where `[data-mode]` applies. See [guides/theming.txt](/llms/guides/theming.txt#customizing-named-tokens).

### Customizing Named Tokens

To replace a named token's value (per mode if needed) or to add brand-new tokens, use the `overrides:` block in `rokkit.config.js`:

```js
// rokkit.config.js
export default {
  overrides: {
    // Reserved-name override — replaces the named-token default
    // via cascade order. Useful when the auto-derived dark-mode
    // shade doesn't read correctly against the dark paper bg.
    'paper-edge': { light: 'kami.400', dark: 'oklch(0.04 0.025 85)' },

    // New custom tokens — emit as `--canvas`, `--canvas-grid`, etc.
    // Color-valued entries also auto-get bg-/text-/border-canvas shortcuts.
    canvas: 'kami.50',
    'canvas-grid': '#d4d4d4'
  }
}
```

Values accept palette refs (`'kami.400'`), raw CSS colors (`'#222222'`, `'oklch(...)'`, `'rgb(...)'`), or `{ light, dark }` objects.

### Legacy z-scale utilities (back-compat)

The older z-scale utilities (`bg-primary-z5`, `text-surface-z7`, …) still resolve as a
back-compat layer pointing at the named tokens above — existing code keeps working, but
new code should use the named-token shortcuts. `tokens: 'extended'` additionally emits
the full 11-shade palette ladder per role (for charts / data-viz).

See `docs/llms/index.txt` for the full named-token guide and all supported prefixes.

### Icon Shortcuts

All Rokkit state icons are registered as shortcut aliases pointing to the semantic icon collection (`i-semantic:{name}`):

```html
<span class="checkbox-checked"></span>   <!-- → i-semantic:checkbox-checked -->
<span class="node-opened"></span>        <!-- → i-semantic:node-opened -->
<span class="action-add"></span>         <!-- → i-semantic:action-add -->
```

When `icons.style` is set (e.g., `'solid'`), shortcuts include the variant suffix:

```html
<span class="action-add"></span>         <!-- → i-semantic:action-add-solid -->
```

Available styles: `undefined` (default), `'solid'`, `'outline'`, `'duotone-outline'`.

The full `DEFAULT_ICONS` list (70+ icons) is also added to the UnoCSS safelist.

### Skin Preflights

Each entry in `options.skins` (other than `default`) generates a `[data-skin='name']`
preflight CSS block. Setting `data-skin='name'` on any ancestor element scopes that palette
to the subtree — the `themable` action writes it to the root element, making it app-wide.

```js
presetRokkit({
  skins: {
    ocean: { surface: 'slate', ink: 'slate', primary: 'sky', accent: 'teal' }
  }
})
```

The preset emits (in the UnoCSS preflight):

```css
/* light block — always emitted for non-default skins */
[data-skin='ocean'] { --paper: oklch(…); --primary: oklch(…); … }

/* dark block — only emitted when the skin has a dual-palette role */
[data-mode='dark'][data-skin='ocean'] { --paper: oklch(…); … }
```

The `default` skin is **not** emitted as a `[data-skin='default']` block — its vars live
in the `:root` preflight, so `data-skin='default'` simply inherits `:root`.

> **Note:** The earlier `skin-{name}` CSS class approach has been removed. Use the
> `data-skin='name'` attribute (written by `themable` / `vibe.skin`) instead.

Built-in skins (`default`, `ocean`, `violet`, `rose`, `emerald`) are exported as
`BUILTIN_SKINS` from `@rokkit/unocss` and merged automatically — consumer `skins` in
`rokkit.config.js` merge on top and can override built-in entries by name.

```js
import { BUILTIN_SKINS } from '@rokkit/unocss'
// { default, ocean, violet, rose, emerald }
```

### Dark Mode

Dark mode uses `:is([data-mode="dark"],[data-mode="dark"] *)` selectors, which match both when `data-mode` is on the same element or an ancestor. This enables programmatic theme switching:

```html
<body data-mode="dark">   <!-- force dark -->
<body data-mode="light">  <!-- force light -->
```

The `dark:` variant in utility classes respects this selector:

```html
<div class="bg-white dark:bg-slate-900">
```

### Font Families

Registers CSS variable–backed font family tokens in the UnoCSS theme. The actual font names are **not baked into the preset** — they are populated at build time from the `typography:` config in `rokkit.config.js`. The preset emits CSS vars (`--font-display`, `--font-ui`, `--font-mono`) and the theme keys point at those vars:

| Token | CSS variable | Populated from `typography:` key |
|-------|-------------|----------------------------------|
| `font-sans` / `font-body` | `var(--font-sans)` → `var(--font-ui)` | `typography.sans` or `typography.ui` |
| `font-heading` | `var(--font-heading)` → `var(--font-display)` | `typography.heading` or `typography.display` |
| `font-mono` | `var(--font-mono)` | `typography.mono` |

Example — set fonts in `rokkit.config.js`:

```js
export default {
  typography: {
    sans: "'Inter', system-ui, sans-serif",
    mono: "'JetBrains Mono', monospace"
  }
}
```

If `typography` is omitted (or a key is `null`), no `--font-*` var is emitted and the browser falls back to inherited fonts.

### Additional Rules

- `hidden` → `display: none` (simple utility alias)

### Svelte Extractor

`extractorSvelte` is included automatically — class names inside Svelte template expressions and `class:` directives are correctly scanned.

### Transformers

- `transformerDirectives` — enables `@apply` and `@screen` in CSS
- `transformerVariantGroup` — enables `hover:(bg-red text-white)` grouped variants

## `presetBackgrounds` — Optional Background Patterns

A separate, opt-in preset for CSS background pattern utilities:

```js
import { presetRokkit, presetBackgrounds } from '@rokkit/unocss'

export default defineConfig({
  presets: [presetRokkit(), presetBackgrounds()]
})
```

Adds classes:

| Class | Description |
|-------|-------------|
| `bg-graph-paper` | Major + minor grid lines |
| `bg-grid-paper` | Simple square grid |
| `bg-ruled-paper` | Horizontal ruled lines |
| `bg-pattern-diagonal` | 45° diagonal lines |
| `bg-pattern-diagonal-reverse` | -45° diagonal lines |
| `bg-pattern-vertical` | Vertical stripes |
| `bg-pattern-horizontal` | Horizontal stripes |
| `bg-pattern-crosshatch` | Crosshatch (diamond mesh) |
| `bg-pattern-dots` | Dot grid |
| `bg-pattern-checker` | Checkerboard |

Pattern appearance is controlled via CSS custom properties: `--pattern-color`, `--pattern-fill`, `--pattern-size`, `--pattern-line`.

## Using `rokkit.config.js`

Place a `rokkit.config.js` at your project root to keep preset options out of your build config. `loadConfig` is called internally by `presetRokkit` and merges the file with inline options:

```js
// rokkit.config.js — simple (z-flip handles dark mode automatically)
export default {
  skin: {
    primary: 'violet',
    surface: 'zinc'     // same palette — shade scale flips for dark mode via base.css
  },
  ...
}
```

```js
// rokkit.config.js — dual-palette (separate light/dark palette per role)
export default {
  palettes: {
    kami: { 50: '0.985 0.005 85', /* … */ 950: '0.170 0.010 50' },
    sumi: { 50: '0.975 0.008 85', /* … */ 950: '0.170 0.010 50' }
  },
  skin: {
    surface: { light: 'kami', dark: 'sumi' },  // explicit dark palette
    primary: 'shu'
  },
  colorSpace: 'oklch',
  skins: {
    // skin IS the default colormap — skins are named alternatives
    ocean:    { surface: 'slate', primary: 'sky' },
    vibrant:  { primary: 'blue', secondary: 'purple' }
  },
  icons: {
    app:       '@rokkit/icons/app.json',
    solar:     '@iconify-json/solar/icons.json'
  },
  themes:     ['rokkit'],
  switcher:   'full',
  storageKey: 'rokkit-theme'
}
```

Then use `presetRokkit()` with no arguments — the config file is auto-loaded.

## themeHook — SvelteKit SSR flash prevention

Imported from `@rokkit/unocss/hooks`. Injects a synchronous inline script into `<body>` that reads persisted theme settings from `localStorage` and applies them as `data-*` attributes before first paint — preventing FOUC (flash of unstyled content). Requires `@rokkit/unocss` ≥ 1.1.12.

```js
// src/hooks.server.ts
import { themeHook } from '@rokkit/unocss/hooks'

export const handle = themeHook({
  storageKey: 'app-theme',   // must match the themable storageKey (default: 'rokkit-theme')
  defaultMode: 'system',     // 'system' (default) | 'light' | 'dark'
  defaultStyle: 'rokkit'     // fallback style if nothing is persisted
})
```

Compose with other hooks via `sequence()`:

```js
import { sequence } from '@sveltejs/kit/hooks'
import { themeHook } from '@rokkit/unocss/hooks'

export const handle = sequence(themeHook(), paraglideHandle)
```

### `themeHook(options?)` options

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `storageKey` | `string` | `'rokkit-theme'` | `localStorage` key for persisted settings |
| `defaultMode` | `string` | `'system'` | Fallback mode when nothing is persisted. `'system'` (and legacy alias `'auto'`) resolves `prefers-color-scheme` pre-paint so the page never renders `data-mode="system"`. Also accepts `'light'` or `'dark'`. |
| `defaultStyle` | `string` | — | Fallback style if nothing is persisted |
| `defaultDensity` | `string` | `'comfortable'` | Fallback density |
| `defaultRadius` | `string` | `'soft'` | Fallback radius |

If you need to compose with your own `transformPageChunk`, use the lower-level `themeInitScript()` export:

```js
import { themeInitScript } from '@rokkit/unocss/hooks'

const initScript = themeInitScript({ storageKey: 'my-key' })
// In transformPageChunk: html.replace(/(<body[^>]*>)/, `$1${initScript}`)
```

See also: [guides/theming.txt](/llms/guides/theming.txt) for the full SSR flash prevention guide.
