# @rokkit/states

> Reactive state primitives for Rokkit components: proxy wrappers, navigation controllers, theme state, messages, and media queries.

This package provides the data and navigation layer that backs all Rokkit UI components. It is rarely used directly in application code — components consume it internally — but it is useful when building custom components or extending existing ones.

## Install

```bash
npm install @rokkit/states
```

## Exports

```javascript
import {
  ProxyItem, LazyProxyItem, BASE_FIELDS,
  ProxyTree, ProxyTable, ProxyTableTree,
  Wrapper, LazyWrapper,
  vibe,
  messages,
  watchMedia, defaultBreakpoints
} from '@rokkit/states'
```

## ProxyItem

A read-only view mapper for a single item. Wraps a raw item (object or primitive) and provides uniform field access through a configurable field mapping.

**When to use:** Components pass `ProxyItem` instances to snippets. If you write a custom component that needs to normalise raw data (including primitives) through a field map, use `ProxyItem`.

### Key concepts

- `#raw` — the original input, never mutated
- Field mapping translates semantic names (`label`, `value`, `icon`, …) to the actual data keys
- Primitives are auto-normalised: `'Hello'` becomes `{ label: 'Hello', value: 'Hello' }` internally
- Control state (`expanded`, `selected`) is `$state`-tracked; if the underlying item has the field, reads/writes pass through to it
- `children` items are auto-wrapped as `ProxyItem` instances via `$derived`, with path-based keys (`'0'`, `'0-1'`, `'0-1-2'`) assigned automatically

### Constructor

```javascript
new ProxyItem(raw, fields?, key?, level?)
```

| Param | Type | Description |
|-------|------|-------------|
| `raw` | `unknown` | Raw item — object or primitive |
| `fields` | `Partial<BASE_FIELDS>` | Field name overrides |
| `key` | `string` | Path-based key (assigned by ProxyTree) |
| `level` | `number` | Nesting depth: 1 = root |

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `label` | `string` | Mapped display text (`item[fields.label]`) |
| `value` | `unknown` | Mapped value (`item[fields.value]`), falls back to raw item |
| `id` | `string` | Stable unique ID — from item's id field or auto-generated |
| `key` | `string` | Path-based key, e.g. `'0'`, `'0-1-2'` |
| `level` | `number` | Nesting depth (1 = root) |
| `original` | `unknown` | The original raw input — never mutated |
| `fields` | `object` | Merged field-mapping config |
| `disabled` | `boolean` | Whether the item is disabled |
| `hasChildren` | `boolean` | True for objects with a non-empty children array |
| `children` | `ProxyItem[]` | Auto-wrapped child proxies (`$derived`) |
| `type` | `'item'|'group'|'separator'|'spacer'` | Item type |
| `expanded` | `boolean` | Expand state (`$state`); writes through to raw item if field present |
| `selected` | `boolean` | Selection state (`$state`); writes through to raw item if field present |

### Methods

| Method | Description |
|--------|-------------|
| `get(fieldName)` | Read any field by semantic or raw name |
| `set(fieldName, value)` | Write to the underlying item through field mapping; triggers children recompute |
| `mutate(fieldOrBatch, value?)` | Write directly to raw item, bypassing field mapping; accepts `(key, value)` or `{ key: value, … }` |

### Usage example

```javascript
// ProxyItem is typically created by ProxyTree, not manually:
const proxy = new ProxyItem({ name: 'Alice', role: 'admin' }, { label: 'name', value: 'role' })

proxy.label    // 'Alice'
proxy.value    // 'admin'
proxy.get('role')  // 'admin' (raw key fallback)
proxy.get('label') // 'Alice' (via field mapping)
proxy.set('name', 'Bob')  // updates underlying item
```

## LazyProxyItem

Extends `ProxyItem` with lazy-loading support. When a `lazyLoad` function is provided, children are fetched on demand.

**When to use:** Use with `LazyWrapper` and tree components that load children asynchronously. Set `children: true` on items as a sentinel to signal they are loadable but not yet loaded.

### Additional properties

| Property | Type | Description |
|----------|------|-------------|
| `loaded` | `boolean` | `false` only for sentinel nodes (`children === true`) that need fetching |
| `loading` | `boolean` | `true` while async fetch is in progress |

### Additional method

| Method | Description |
|--------|-------------|
| `fetch()` | Async. Calls `lazyLoad(value, original)`, writes results via `set('children', …)`, sets `loaded = true`. No-op if already loaded or loading. |

### Constructor

```javascript
new LazyProxyItem(raw, fields?, key?, level?, lazyLoad?)
```

`lazyLoad` is `async (value, raw) => children[]`.

## ProxyTree

Manages a tree of `ProxyItem` instances. Derives a reactive flat view for rendering and a lookup map for O(1) access.

**When to use:** The data layer behind `Wrapper`. Use directly only when building custom tree-based components.

### Constructor

```javascript
new ProxyTree(items?, fields?, options?)
```

`options.createProxy` — factory function `(raw, fields, key, level) => ProxyItem` for custom proxy types (e.g. `LazyProxyItem`).

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `flatView` | `{ key, proxy, level, hasChildren, isExpandable, type, lineTypes }[]` | Reactive flat render list — re-derives on expand/children changes |
| `roots` | `ProxyItem[]` | Root proxy array |
| `lookup` | `Map<string, ProxyItem>` | All proxies by key (reactive, includes non-expanded nodes) |

### Methods

| Method | Description |
|--------|-------------|
| `append(items)` | Add new root items; creates proxies continuing from current length |
| `addChildren(proxy, items)` | Add raw children to an existing proxy node |

## Wrapper

Navigation controller for persistent list/tree/sidebar components. Owns `focusedKey` state and provides movement, selection, expand/collapse, and typeahead logic on top of a `ProxyTree`.

**When to use:** The navigation layer behind `List` and `Tree` components. Use directly only when building custom persistent-list components. Dropdown variants (`Select`, `Menu`) extend this class.

### Constructor

```javascript
new Wrapper(proxyTree, { onselect?, onchange?, collapsible? }?)
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `onselect` | `Function` | — | Called on every leaf selection with `(value, proxy)` |
| `onchange` | `Function` | — | Called on leaf selection only when value changes |
| `collapsible` | `boolean` | `true` | When `false`, `toggle()`, `collapse()` (closing), and `select()` on groups are no-ops — groups stay permanently open |

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `flatView` | array | Re-derived from proxyTree's flatView |
| `focusedKey` | `string\|null` | Currently focused item key (`$state`) |
| `lookup` | `Map` | Delegated from proxyTree |
| `proxyTree` | `ProxyTree` | The underlying data layer |

### Navigation methods

| Method | Description |
|--------|-------------|
| `next(path)` | Move focus to next navigable item (skips separators/spacers/disabled) |
| `prev(path)` | Move focus to previous navigable item |
| `first(path)` | Focus first navigable item |
| `last(path)` | Focus last navigable item |
| `expand(path)` | Expand focused group; if already open, advances focus to first child |
| `collapse(path)` | Collapse focused group; if leaf/collapsed, moves focus to parent. When `collapsible=false`, always moves to parent without closing |
| `moveTo(path)` | Sync focused key to path — called by Navigator on focusin |
| `moveToValue(v)` | Sync focus to the item matching a semantic value |

### Selection methods

| Method | Description |
|--------|-------------|
| `select(path)` | Select item at path (or `focusedKey`). Groups toggle expanded (only when `collapsible=true`); leaves fire `onchange`/`onselect` |
| `toggle(path)` | Toggle group expansion — does not fire `onselect`. No-op when `collapsible=false` |
| `findByText(query, startAfterKey?)` | Typeahead: returns key of first item whose label starts with query (case-insensitive, wraps around) |
| `cancel(path)` | No-op on persistent lists; override in dropdown subclasses to close |
| `blur()` | No-op on persistent lists; override in dropdown subclasses |

## LazyWrapper

Extends `Wrapper` with lazy-loading support. Intercepts `expand()`, `select()`, and `toggle()` on unloaded sentinel nodes to call `proxy.fetch()` before delegating to the base behavior.

**When to use:** Backing layer for tree components with on-demand child loading.

### Constructor

```javascript
new LazyWrapper(proxyTree, { onselect?, onchange?, onlazyload? }?)
```

`onlazyload` is `async () => items[]` — called by `loadMore()` for root-level pagination.

### Additional method

| Method | Description |
|--------|-------------|
| `loadMore()` | Async. Calls `onlazyload()` and appends results to the proxy tree root. |

## ProxyTable

Tabular data layer. Extends `ProxyTree` and adds columns + sort state. The same `Wrapper` and `Navigator` that drive lists/trees can navigate over a `ProxyTable` without modification — it just has a different shape on its data layer.

**When to use:** Backing data for the `Table` component, or whenever you need flat tabular rows with sortable columns and reactive selection.

### Constructor

```javascript
new ProxyTable(data?, options?)
// options: { columns?, fields?, onsort? }
```

Columns are auto-derived from `data` when not provided. Each column gets `{ sortable: true, sorted: 'none' }` defaults.

### Properties

| Property | Type | Description |
|----------|------|-------------|
| `columns` | `object[]` | Column definitions with `name`, `sortable`, `sorted` (`$state`) |
| `sortState` | `{ column, direction }[]` | Active sort stack (`$state`) |
| `flatView` | array | Inherited from ProxyTree (`$derived`) |
| `lookup` | `Map` | Inherited from ProxyTree (`$derived`) |

### Sort methods

| Method | Description |
|--------|-------------|
| `sortBy(columnName, extend?)` | Toggle sort on a column. Cycles `none → ascending → descending → none`. `extend=true` (Shift+click) adds to sort stack |
| `clearSort()` | Clear all sort state; restore original data order |
| `update(data)` | Replace underlying rows; re-applies any active sort |
| `updateColumns(columns)` | Replace column definitions; preserves sorted state for surviving columns |

### Hooks

- `_compareRows(a, b)` and `_sortedData(rows)` are protected and can be overridden by subclasses (used by `ProxyTableTree` for hierarchy-aware sort).

## ProxyTableTree

Hierarchical analog of ProxyTable. Accepts nested rows (each row may carry `children: []`) and sorts siblings within each parent so the parent/child structure survives any sort.

**When to use:** Backing data for the `TreeTable` component, or whenever you need a sortable grid over hierarchical rows.

### Constructor

```javascript
new ProxyTableTree(data?, options?)
// options: { columns?, fields?, onsort? }
//   fields.children customizes the child-array field (default: 'children')
```

### Helpers for flat shapes

Use `nestByPath` / `nestByColumns` from `@rokkit/data` to convert flat shapes into the nested form ProxyTableTree expects.

```javascript
import { nestByPath, nestByColumns } from '@rokkit/data'

const nestedFromPath = nestByPath(rows, { column: 'path' })
const nestedFromGroups = nestByColumns(rows, ['region', 'country'])
```

## vibe

Singleton reactive store for the current theme state. Components and actions read from `vibe` to apply palette, mode, density, and direction.

**When to use:** Read `vibe` to react to theme changes. Write to its properties to change the theme programmatically.

### Properties (all reactive `$state`)

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `mode` | `'light'\|'dark'` | `'dark'` | Color scheme |
| `density` | `'cozy'\|'compact'\|'comfortable'` | `'comfortable'` | Spacing density |
| `style` | `string` | `'rokkit'` | Active theme style name |
| `skin` | `string` | `'default'` | Active named skin. The setter rejects values not in `allowedSkins` and no-ops when the value matches the current skin. The `themable` action writes this as `data-skin` onto the root element. |
| `direction` | `'ltr'\|'rtl'` | auto-detected | Text direction |
| `isRTL` | `boolean` | — | Convenience flag for RTL |
| `palette` | `object` | — | Derived color rules from `colorMap` + `colors` (`$derived`) |
| `colors` | `object` | — | Base color definitions |
| `colorMap` | `object` | — | Maps semantic color roles to base color keys |
| `allowedStyles` | `string[]` | `DEFAULT_STYLES` | Whitelist of valid style values |
| `allowedSkins` | `string[]` | `['default']` | Whitelist of valid skin values. Set this before assigning `vibe.skin` so the new value is accepted. Used by `SkinSwitcherToggle` as its default option list. |

### Methods

| Method | Description |
|--------|-------------|
| `update({ style?, mode?, density?, direction? })` | Batch update multiple theme properties |
| `load(key)` | Load and apply theme from `localStorage` |
| `save(key)` | Persist current theme (`style`, `mode`, `density`, `direction`, `skin`) to `localStorage` |
| `detectDirection()` | Re-detect direction from the document's lang attribute |

### Usage example

```javascript
import { vibe } from '@rokkit/states'

// Read
console.log(vibe.mode)     // 'dark'
console.log(vibe.density)  // 'comfortable'
console.log(vibe.skin)     // 'default'

// Write
vibe.mode = 'light'
vibe.density = 'compact'

// Skins — set allowedSkins first so the setter accepts the value
vibe.allowedSkins = ['default', 'ocean', 'violet', 'rose', 'emerald']
vibe.skin = 'ocean'

// Persist (saves style, mode, density, direction, skin)
vibe.save('myapp-theme')
// Later:
vibe.load('myapp-theme')
```

## messages

Singleton reactive store for localised UI strings. All Rokkit components read directly from `messages` fields for their labels and ARIA strings.

**When to use:** Override default English strings for internationalisation or to customise button/label text in components.

### Properties

All string fields are direct `$state` on the `messages` instance — there is no `.current` indirection:

| Property | Type | Description |
|----------|------|-------------|
| `emptyList` | `string` | Shown when a list has no items |
| `emptyTree` | `string` | Shown when a tree has no data |
| `loading` | `string` | Loading placeholder text |
| `noResults` | `string` | Shown when a search/filter returns nothing |
| `select` | `string` | Default select placeholder |
| `search` | `string` | Search input placeholder |
| `list` | `object` | `{ label }` |
| `tree` | `object` | `{ label, expand, collapse, loading, loadMore }` |
| `toolbar` | `object` | `{ label }` |
| `menu` | `object` | `{ label }` |
| `table` | `object` | `{ empty, sortAscending, sortDescending }` |
| `command` | `object` | `{ placeholder, noResults, label }` |
| (and more…) | | See default keys list below |

### Methods

| Method | Description |
|--------|-------------|
| `register(locale, overrides)` | Register translation overrides for a locale. Unspecified keys fall back to English defaults. |
| `setLocale(locale)` | Switch to a previously registered locale. Use `'en'` to restore English defaults. |
| `set(custom)` | Deep-merge custom strings over defaults without naming a locale. Partial updates are fine — only supplied keys are replaced. |
| `reset()` | Restore all defaults and clear all registered locales |

### `locale` getter

```javascript
messages.locale  // e.g. 'en', 'de', '_custom'
```

### Default message keys

`emptyList`, `emptyTree`, `loading`, `noResults`, `select`, `search`, `list`, `tree`, `toolbar`, `menu`, `toggle`, `rating`, `stepper`, `breadcrumbs`, `carousel`, `tabs`, `table`, `code`, `range`, `search_`, `filter`, `grid`, `uploadProgress`, `floatingNav`, `mode`, `command`

### Usage example

```javascript
import { messages } from '@rokkit/states'

// Partial override (merges with defaults)
messages.set({
  emptyList: 'Aucun élément trouvé',
  tree: { expand: 'Développer', collapse: 'Réduire' },
  select: 'Choisir une option'
})

// Read in a component — access directly, no .current
messages.emptyList  // 'Aucun élément trouvé'
messages.command.placeholder  // 'Run a command…'

// i18n: register a locale and switch to it
messages.register('de', {
  emptyList: 'Keine Einträge',
  select: 'Option auswählen'
})
messages.setLocale('de')
messages.locale  // 'de'
```

## watchMedia

Creates a record of reactive media query matchers using Svelte's `MediaQuery`. Each entry is `true` when the query matches.

**When to use:** Responsive layouts that need to react to breakpoint or preference changes inside Svelte components.

### Signature

```javascript
function watchMedia(breakpoints?: Record<string, string>): Record<string, MediaQuery>
```

Returns an object where each key holds a `MediaQuery` instance — read `.matches` (or use in reactive context) to get the current boolean state.

### defaultBreakpoints

```javascript
{
  small:        '(max-width: 767px)',
  medium:       '(min-width: 768px) and (max-width: 1023px)',
  large:        '(min-width: 1024px)',
  extraLarge:   '(min-width: 1280px)',
  short:        '(max-height: 399px)',
  landscape:    '(orientation: landscape)',
  tiny:         '(orientation: portrait) and (max-height: 599px)',
  dark:         '(prefers-color-scheme: dark)',
  noanimations: '(prefers-reduced-motion: reduce)'
}
```

### Usage example

```svelte
<script>
  import { watchMedia, defaultBreakpoints } from '@rokkit/states'

  const media = watchMedia(defaultBreakpoints)
</script>

{#if media.small.matches}
  <MobileNav />
{:else}
  <SidebarNav />
{/if}
```

Custom breakpoints:

```javascript
const media = watchMedia({
  phone: '(max-width: 600px)',
  tablet: '(min-width: 601px) and (max-width: 1024px)'
})
```

## commands

App-wide command registry. Defines commands (keyboard shortcuts + labelled
actions) as a singleton that both the `shortcuts` action and `CommandPalette`
read from. See [Commands & Shortcuts guide](/llms/guides/commands.txt).

### Command shape

```javascript
/** @typedef {Object} Command
 * @property {string}        id        — unique key; re-registering same id replaces + warns
 * @property {string}        label     — display text shown in CommandPalette (consumer-localised)
 * @property {() => void}    run       — called on shortcut match or palette select
 * @property {string}        [shortcut]  — e.g. 'mod+k' | 'mod+shift+p' | '?'
 * @property {string}        [group]     — palette category header
 * @property {boolean}       [global]    — fire even while text input is focused (default false)
 * @property {() => boolean} [enabled]   — optional gate; returning false suppresses execution
 * @property {string[]}      [keywords]  — extra palette search terms
 */
```

`mod` resolves to `Cmd` on macOS and `Ctrl` everywhere else.

### Registry API

| Method / Property | Signature | Description |
|-------------------|-----------|-------------|
| `register(cmd)` | `(Command) => () => void` | Register one command; returns unregister fn |
| `registerMany(cmds)` | `(Command[]) => () => void` | Register a batch; returns single unregister fn |
| `unregister(id)` | `(string) => void` | Remove a command by id |
| `execute(id)` | `(string) => void` | Run a command by id (checks `enabled` gate) |
| `resolve(event)` | `(KeyboardEvent) => Command\|null` | Match a keyboard event to a command |
| `all` | `Command[]` | Reactive array of all registered commands (`$state`) |

### Usage

```svelte
<script>
  import { commands } from '@rokkit/states'
  import { onMount } from 'svelte'

  let paletteOpen = $state(false)

  onMount(() => {
    return commands.registerMany([
      {
        id: 'palette.open',
        label: 'Open command palette',
        shortcut: 'mod+k',
        global: true,
        group: 'general',
        run: () => { paletteOpen = true }
      }
    ])
  })
</script>
```

### normalizeShortcut / eventToShortcut

Two utility functions exported alongside `commands`:

```javascript
import { normalizeShortcut, eventToShortcut } from '@rokkit/states'

normalizeShortcut('mod+K')      // 'meta+k' on macOS, 'ctrl+k' elsewhere
eventToShortcut(keyboardEvent)  // canonical string built from a live KeyboardEvent
```

Use these when building custom shortcut display UIs or comparing shortcut strings.

### messages.command

`CommandPalette` reads its chrome strings from `messages.command` directly:

| Key | Default |
|-----|---------|
| `messages.command.placeholder` | `'Run a command…'` |
| `messages.command.noResults` | `'No commands found'` |
| `messages.command.label` | `'Command palette'` |

Override via `messages.set({ command: { … } })` or `messages.register('locale', { command: { … } })`.
