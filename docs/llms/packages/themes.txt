# @rokkit/themes

> CSS theme files for Rokkit components — structural base styles and visual theme variants.

## Install

```bash
npm install @rokkit/themes
```

## Available Themes

Rokkit ships 11 visual themes. Each is a separate CSS file:

| Theme | Import path | Personality |
|-------|-------------|-------------|
| **rokkit** | `@rokkit/themes/rokkit.css` | Rich gradients, glowing borders, and vibrant accents. The default theme. |
| **minimal** | `@rokkit/themes/minimal.css` | Clean, understated styling with subtle borders and soft backgrounds. |
| **material** | `@rokkit/themes/material.css` | Material Design inspired with elevation shadows and depth. |
| **frosted** | `@rokkit/themes/frosted.css` | Frosted glass with backdrop blur, transparency, and subtle borders. |
| **shadcn** | `@rokkit/themes/shadcn.css` | Flat borders and ring focus — shadcn/ui-inspired. |
| **daisy-ui** | `@rokkit/themes/daisy-ui.css` | Rounded-full buttons and bold fills — DaisyUI-inspired. |
| **bits-ui** | `@rokkit/themes/bits-ui.css` | Rounded-lg and polished shadow-sm — Bits UI-inspired. |
| **carbon** | `@rokkit/themes/carbon.css` | Square corners and bottom-border inputs — IBM Carbon-inspired. |
| **ant-design** | `@rokkit/themes/ant-design.css` | Thin borders and dense layout — Ant Design-inspired. |
| **grada-ui** | `@rokkit/themes/grada-ui.css` | Coral/purple gradient identity. |
| **base** | `@rokkit/themes/base.css` | Structural styles only (layout, positioning — no visual treatment). |

## CSS Import Structure

Every Rokkit application needs two imports: the **base** file (structural layout) and at least one **theme** file (colors and visual effects).

```css
/* In your global CSS (e.g., app.css or +layout.svelte <style>) */
@import '@rokkit/themes/base.css';    /* Required — structural layout for all components */
@import '@rokkit/themes/rokkit.css';  /* One or more visual themes */
```

Or import all themes at once (larger bundle, but lets you switch themes at runtime):

```css
@import '@rokkit/themes/base.css';
@import '@rokkit/themes/rokkit.css';
@import '@rokkit/themes/minimal.css';
@import '@rokkit/themes/material.css';
@import '@rokkit/themes/frosted.css';
@import '@rokkit/themes/shadcn.css';
/* ... etc. */
```

### Per-component imports (tree-shaking)

Each theme directory also has per-component CSS source files. Import only what you use:

```css
@import '@rokkit/themes/base/button.css';
@import '@rokkit/themes/rokkit/button.css';
```

## Activating a Theme with `data-style`

The `rokkit` theme is active by default with no attribute needed. Other themes are activated by setting `data-style` on any ancestor element — typically `<body>` or a layout wrapper:

```html
<!-- Minimal theme for the whole page -->
<body data-style="minimal">

<!-- Material theme for a specific section -->
<div data-style="material">
  ...components...
</div>
```

Multiple themes can coexist on the same page — components inherit the nearest ancestor's `data-style`.

## Color Palette and Custom Colors

The base CSS file sets all semantic color variables (`--color-primary`, `--color-surface`, etc.) using the default palette (orange primary, slate surface, etc.).

To change these colors without rebuilding CSS, override the CSS variables directly:

```css
:root {
  --color-primary:     76,29,149;   /* violet-900 in r,g,b format */
  --color-primary-500: 109,40,217;
  /* ... shades 50–950 */
}
```

For a structured approach, use `rokkit.config.js` with `@rokkit/unocss`. The config drives both the UnoCSS utility classes and the CSS variable values injected at build time:

```js
// rokkit.config.js
export default {
  colors: {
    primary: 'violet',
    surface: 'zinc',
    accent:  'sky'
  }
}
```

See `docs/llms/packages/unocss.txt` for the full `rokkit.config.js` API and how `presetRokkit` consumes it.

## How Themes Relate to Components

Rokkit components are **unstyled by default** — they only have structural CSS (`data-*` attribute selectors for layout, sizing, and behavior). Visual appearance comes entirely from the imported theme files.

The separation means:
- Components work without any theme (plain, minimal structure)
- Swapping theme files changes the look without touching component code
- `data-style` on an ancestor scopes a theme to a subtree

Each theme targets Rokkit's data attributes. For example, `[data-button]` gets border, background, and hover effects from the active theme file.

## Data-Attribute Hooks for Custom Styling

Rokkit components expose state through `data-*` attributes instead of class names. Target these in your CSS to style component and state variations:

```css
/* Target the list container */
[data-list] { ... }

/* Target individual items */
[data-list-item] { ... }

/* Selected state */
[data-list-item][data-selected] {
  background: var(--primary-100);
  color: var(--primary-800);
}

/* Keyboard-focused state */
[data-list-item][data-focused] {
  outline: 2px solid var(--primary-500);
  outline-offset: -2px;
}
```

### Common State Attributes

These state attributes are applied consistently across all components:

| Attribute | When Present | Components |
|-----------|-------------|-----------|
| `data-selected` | Item is selected | List, Select, Tabs, Toggle |
| `data-focused` | Item has keyboard focus | All navigable components |
| `data-disabled` | Item is disabled | All |
| `data-expanded` | Group or node is expanded | List, Tree, Select |
| `data-open` | Dropdown is open | Select, Menu |

### Building a Custom Theme

Create a theme CSS file that targets data attributes, then import it after the base CSS:

```css
/* my-theme.css */
[data-list-item] {
  border-radius: 0.5rem;
  transition: background 150ms ease;
}

[data-list-item][data-selected] {
  background: var(--primary-100);
  color: var(--primary-800);
}

[data-list-item][data-focused] {
  outline: 2px solid var(--primary-500);
  outline-offset: -2px;
}
```

```css
/* app.css */
@import '@rokkit/themes/base.css';
@import './my-theme.css';
```

Activate it with `data-style` just like a built-in theme:
```html
<body data-style="my-theme">
```

## Skins

Skins are pre-built color bundles applied via a CSS class on any ancestor element. They layer a specific color palette on top of the active theme, without changing the structural or visual style:

```svelte
<!-- Applied to any container or wrapper -->
<div class="skin-default">...</div>
<div class="skin-vibrant">...</div>
<div class="skin-seaweed">...</div>
```

Skins are useful for scoping different color palettes to different sections of a page without changing the overall theme.

**Skin vs. `data-style`:**
- `data-style` selects the visual personality (gradients, shadows, border radius, transitions)
- Skins change the color palette within that personality
