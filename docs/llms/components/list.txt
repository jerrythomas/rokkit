# Rokkit List Component

> Data-driven, accessible list with keyboard navigation, selection, and collapsible groups. Built on the ProxyItem + Wrapper + Navigator stack.

List renders an array as a navigable `<nav>`. **It is data-first: you describe items in data, and the component renders icon, label, description, badge, avatar, and shortcut automatically.** Reach for snippets only when the *markup* must differ from the default — not to add an icon or a badge (those are data).

## Usage tiers — use the SIMPLEST that works

Most lists need Tier 0 or 1. Going straight to `itemContent` is the most common mistake — it throws away the built-in rendering and you re-implement icon/label/badge by hand for every consumer.

**Tier 0 — `<List {items}>`** — primitives, or objects whose keys already match the defaults (`label`, `icon`, `value`, `badge`, `description`, …). No config.
```svelte
<List items={['Dashboard', 'Reports', 'Settings']} bind:value />

<List items={[
  { label: 'Inbox',   icon: 'i-lucide:inbox', badge: '12' },
  { label: 'Starred', icon: '⭐', description: 'Flagged messages' }
]} bind:value />
```

**Tier 1 — `<List {items} {fields}>`** — same, but your data uses different key names. `fields` is a read-only remap; it never mutates your data.
```svelte
<List items={routes} fields={{ label: 'name', value: 'path', icon: 'symbol' }} bind:value />
```

**Tier 2 — per-item / per-group named snippets** — when *a few* rows need bespoke markup. Set `item.snippet = 'name'`; every other row keeps the default rendering. Works for **group headers and leaf items alike**.
```svelte
<List {items}>
  {#snippet observatoryHeader(proxy)}…{/snippet}   <!-- only rows with snippet:'observatoryHeader' -->
</List>
```

**Tier 3 — `itemContent` / `groupContent`** — blanket custom rendering for *every* item/group. Last resort; you now own icon/label/badge layout yourself.

> **Before writing any snippet, ask: is this just an icon / badge / description / avatar / second line?** If yes, it's data — use Tier 0/1. Snippets are for structurally different markup (an inline control, a chart, a custom layout).

## What the default item renders (Tiers 0–1, no snippet)

The built-in item renderer (`ItemContent`) outputs, in order, whatever fields are present:

| Field (data key) | Renders as | Notes |
|---|---|---|
| `image` (avatar) | `<img data-item-avatar>` | takes precedence over `icon` |
| `icon` | `<span data-item-icon class={icon}>` **or** `<span data-item-icon-literal>` | a CSS icon class (`i-lucide:home`) renders as an icon; **any other string** (a single character, an emoji like `⭐`, `列`) renders as literal text. So "1-char text vs icon" is the same field — no snippet needed either way. |
| `label` | `<span data-item-label>` | the display text |
| `description` | `<span data-item-description>` | secondary subtitle line, under the label |
| `badge` | `<span data-item-badge>` | trailing count / status pill |
| `shortcut` | `<kbd data-item-shortcut>` | trailing keyboard hint |

All of these are data-driven. You do **not** need a snippet to show an icon, a badge, a description, or an avatar.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `unknown[]` | `[]` | Data array (objects or primitives) |
| `fields` | `Record<string,string>` | `{}` | Remap data keys to component fields |
| `value` | `unknown` | — | Selected value — use `bind:value` |
| `size` | `'sm'\|'md'\|'lg'` | `'md'` | Size variant |
| `disabled` | `boolean` | `false` | Disable all items |
| `collapsible` | `boolean` | `false` | Allow groups to be expanded/collapsed |
| `icons` | `{ opened?, closed? }` | — | Override expand/collapse icons |
| `class` | `string` | `''` | Additional CSS classes on the root `<nav>` |
| `onselect` | `(value, proxy) => void` | — | Called when an item is selected |

## Field mapping (complete)

Default field names the component reads from your data. Override only the ones that differ.

| Field | Default data key | Renders |
|-------|------------------|---------|
| `label` | `'label'` | display text |
| `value` | `'value'` | selection value (falls back to the raw item) |
| `icon` | `'icon'` | icon class **or** literal char/emoji |
| `image` (avatar) | `'image'` | avatar `<img>` |
| `description` | `'description'` | secondary subtitle line |
| `badge` | `'badge'` | trailing badge/count |
| `shortcut` | `'shortcut'` | trailing `<kbd>` hint |
| `href` | `'href'` | renders item as `<a>` (gets `aria-current="page"` when active) |
| `children` | `'children'` | nested items array → group |
| `type` | `'type'` | `'separator'` / `'spacer'` structural markers |
| `disabled` | `'disabled'` | disabled state |
| `expanded` | `'expanded'` | initial expand state for a group |
| `snippet` | `'snippet'` | per-item/group named-snippet override (Tier 2) |

```svelte
<List {items} fields={{ label: 'name', value: 'path', description: 'summary' }} bind:value />
```

## Grouped items / collapsible sections

Items with a `children` array become collapsible group headers. Add `collapsible` to allow toggling; groups start collapsed unless `expanded: true`.

```svelte
<List items={[
  { label: 'Mail', expanded: true, children: [
    { label: 'Inbox', value: 'inbox', badge: '8' },
    { label: 'Drafts', value: 'drafts' }
  ]},
  { label: 'Settings', children: [{ label: 'Profile', value: 'profile' }] }
]} collapsible bind:value />
```

## Tier 2 — per-item / per-group named snippets

Set `item.snippet = 'name'` to route *specific* rows to a named snippet; all other rows keep the default rendering. The named snippet **wins over both `itemContent` (leaves) and `groupContent` (group headers)** — so you can give one group header a custom layout while leaving the rest default:

```svelte
<script>
  const items = [
    { label: 'Observatory', snippet: 'observatoryHeader', children: [{ label: 'Telescopes' }] },
    { label: 'Settings', children: [{ label: 'General' }] },   // default group header
    { label: 'Pinned', snippet: 'pinned' },                    // custom leaf
    { label: 'Reports' }                                       // default leaf
  ]
</script>

<List {items} collapsible={false}>
  {#snippet observatoryHeader(proxy)}
    <span data-item-label>{proxy.label}</span>
    <span class="flex-1"></span>
    <button onclick={(e) => { e.stopPropagation(); /* custom toggle */ }}>Custom Toggle</button>
  {/snippet}

  {#snippet pinned(proxy)}
    <span class="i-lucide:pin"></span>
    <span>{proxy.label}</span>
  {/snippet}
</List>
```

`proxy.snippet` is read via the field accessor (it honors `fields.snippet`), exactly like `icon`/`href`.

## Tier 3 — itemContent / groupContent (blanket custom)

Replaces the inner content of **every** leaf / group. The component still renders the `<button>`/`<a>` wrapper, `data-path`, and navigation. Use only when the default renderer genuinely can't express the layout.

```svelte
<List {items} collapsible bind:value>
  {#snippet itemContent(proxy)}
    {#if proxy.icon}<span class={proxy.icon} aria-hidden="true"></span>{/if}
    <span class="flex-1">{proxy.label}</span>
    {#if proxy.get('badge')}<span class="text-xs text-ink-mute">{proxy.get('badge')}</span>{/if}
  {/snippet}
  {#snippet groupContent(proxy)}
    <span class={proxy.icon}></span>
    <span>{proxy.label}</span>
  {/snippet}
</List>
```

### Interactive elements inside snippets

`stopPropagation()` so the Navigator doesn't also fire selection. Do NOT add `onclick` on an element that carries `data-path` (the wrapper) — the Navigator already handles it.

```svelte
{#snippet itemContent(proxy)}
  <span class="flex-1">{proxy.label}</span>
  <input type="checkbox" checked={proxy.get('checked')}
    onchange={(e) => { proxy.value.checked = e.currentTarget.checked }}
    onclick={(e) => e.stopPropagation()} />
{/snippet}
```

## ProxyItem API (passed to all content snippets)

| Property / method | Description |
|---|---|
| `proxy.label` | mapped display text |
| `proxy.icon` | mapped icon class / literal |
| `proxy.href` | mapped href |
| `proxy.value` | the original raw item (object or primitive) |
| `proxy.disabled` | whether this item is disabled |
| `proxy.expanded` | expand state for groups — reactive `$state` |
| `proxy.hasChildren` | true when the item has a children array |
| `proxy.get('key')` | read any field by semantic or raw name (`proxy.get('badge')`, `proxy.get('description')`) |

## Theming

### Data attributes

Outer structure:

| Attribute | Element |
|---|---|
| `data-list` | root `<nav>` (also carries `data-size`, `data-collapsible`, `data-disabled`) |
| `data-list-item` | leaf `<button>`/`<a>` (also `data-active`, `data-disabled`, `data-level`) |
| `data-list-group` | group header `<button>` (also `aria-expanded`, `data-level`) |
| `data-list-separator` | `<hr>` |
| `data-list-spacer` | `<div>` |

Inner (default-rendered) content — target these to restyle the built-in rendering without a snippet:

| Attribute | Element |
|---|---|
| `data-item-avatar` | avatar `<img>` |
| `data-item-icon` / `data-item-icon-literal` | icon-class span / literal-char span |
| `data-item-text` | label + description wrapper |
| `data-item-label` | label text |
| `data-item-description` | secondary line |
| `data-item-badge` | trailing badge |
| `data-item-shortcut` | trailing `<kbd>` |

### Styling via the `class` prop (UnoCSS arbitrary variants)

`class` lands on the root `<nav data-list>`. Use UnoCSS arbitrary variants to reach descendants — `&` is the root, `_` is a descendant combinator, and the **whole selector must be wrapped in `[...]`**:

```svelte
<List {items} collapsible class="
  [&_[data-list-group]]:px-4
  [&_[data-list-item]]:py-1
  [&_[data-item-badge]]:text-xs [&_[data-item-badge]]:text-ink-mute
" />
```

(`[&>[data-list-group]]:px-4` with `>` targets direct children only; prefer `_` (descendant) unless you specifically need the direct-child combinator.)

Or theme globally with named tokens in your theme CSS:

```css
[data-list-item][data-active] { background: var(--accent-soft); color: var(--accent); }
[data-list-group] { color: var(--ink-mute); font-weight: 600; }
```

## Keyboard navigation

`↑`/`↓` move focus · `→`/`←` expand/collapse groups (when `collapsible`) · `Home`/`End` · `Enter`/`Space` select or toggle · `a-z` typeahead. Disabled items are skipped.

```javascript
import { List } from '@rokkit/ui'
```
