# Rokkit Tabs Component

> Data-driven tabbed interface with keyboard navigation, orientation variants, editable tabs, and full panel customisation. Built on the ProxyItem + Wrapper + Navigator stack.

Tabs renders a list of tab triggers linked to content panels. **It is data-first: you describe tabs in data (label, icon, content), and the component renders triggers and panels automatically.** Reach for snippets only when the *markup* must differ from the default ŌĆö not to change a label or add an icon (those are data).

## Usage tiers ŌĆö use the SIMPLEST that works

Most tab sets need Tier 0 or 1. Going straight to `itemContent` or `tabPanel` is the most common mistake ŌĆö it throws away the built-in rendering and you re-implement icon/label by hand.

**Tier 0 ŌĆö `<Tabs {options}>`** ŌĆö objects whose keys already match the defaults (`label`, `value`, `icon`, `content`, ŌĆ”). No config.
```svelte
<Tabs options={[
  { value: 'overview', label: 'Overview', content: 'Overview content hereŌĆ”' },
  { value: 'api',      label: 'API',      content: 'API docs hereŌĆ”',   icon: 'i-lucide:code' },
  { value: 'examples', label: 'Examples', content: 'Examples hereŌĆ”' }
]} bind:value />
```

**Tier 1 ŌĆö `<Tabs {options} {fields}>`** ŌĆö same, but your data uses different key names. `fields` is a read-only remap; it never mutates your data.
```svelte
<Tabs options={pages} fields={{ label: 'title', value: 'id', content: 'body' }} bind:value />
```

**Tier 2 ŌĆö per-item named snippets** ŌĆö when *a few* triggers need bespoke markup. Set `item.snippet = 'name'`; every other trigger keeps the default rendering.
```svelte
<Tabs {options}>
  {#snippet special(proxy, selected)}ŌĆ”{/snippet}   <!-- only items with snippet:'special' -->
</Tabs>
```

**Tier 3 ŌĆö `itemContent` / `tabPanel`** ŌĆö blanket custom rendering for *every* trigger or every panel. Last resort; you now own trigger icon/label or panel layout yourself.

> **Before writing any snippet, ask: is this just an icon / label / content string?** If yes, it's data ŌĆö use Tier 0/1. Snippets are for structurally different markup (a chart panel, a rich trigger badge, a custom layout).

## What the default renders (Tiers 0ŌĆō1, no snippet)

### Default trigger (`data-tabs-trigger`)

The built-in trigger renderer outputs, in order, whatever fields are present:

| Field (data key) | Renders as | Notes |
|---|---|---|
| `icon` | `<span data-tabs-icon class={icon}>` | a CSS icon class (`i-lucide:code`) applied to a span |
| `label` | `<span data-tabs-label>` | the display text; also used as `aria-label` |

### Default panel (`data-tabs-panel`)

The built-in panel renderer outputs:

| Field (data key) | Renders as | Notes |
|---|---|---|
| `content` | `<div data-tabs-content>` | raw string/HTML content of the panel |

`content` is accessed as a raw data key (not in BASE_FIELDS); the field alias is `'content'` by default and is overridable via `fields`.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `options` | `unknown[]` | `[]` | Data array (objects) |
| `fields` | `Record<string,string>` | `{}` | Remap data keys to component fields |
| `value` | `unknown` | ŌĆö | Selected tab value ŌĆö use `bind:value` |
| `orientation` | `'horizontal'\|'vertical'` | `'horizontal'` | Tab list direction |
| `position` | `'before'\|'after'` | `'before'` | Tab list placement relative to panels (top/left vs bottom/right) |
| `align` | `'start'\|'center'\|'end'` | `'start'` | Alignment of tabs within the trigger list |
| `name` | `string` | `'tabs'` | Accessible `aria-label` for the tablist |
| `editable` | `boolean` | `false` | Show add/remove controls |
| `placeholder` | `string` | from messages | Text shown when `value` is `undefined` |
| `disabled` | `boolean` | `false` | Disable all tabs |
| `class` | `string` | `''` | Additional CSS classes on the root element |
| `onchange` | `(value, proxy) => void` | ŌĆö | Called when selection changes |
| `onselect` | `(value, proxy) => void` | ŌĆö | Called on every click (including same tab) |
| `onadd` | `() => void` | ŌĆö | Called when add button clicked (editable mode) |
| `onremove` | `(value) => void` | ŌĆö | Called when remove button clicked; receives the tab's value |

## Field mapping (complete)

Default field names the component reads from your data. Override only the ones that differ.

| Semantic key | Default data key | Renders |
|---|---|---|
| `label` | `'label'` | trigger display text and `aria-label` |
| `value` | `'value'` | tab selection value |
| `icon` | `'icon'` | icon class string for the trigger |
| `disabled` | `'disabled'` | disabled state for individual tabs |
| `snippet` | `'snippet'` | per-item named-snippet override (Tier 2) |
| `content` | `'content'` | panel body (raw key, not in BASE_FIELDS ŌĆö overridable via `fields`) |

```svelte
<Tabs options={pages} fields={{ label: 'title', value: 'slug', content: 'body', icon: 'sym' }} bind:value />
```

## Layout variants

```svelte
<!-- Vertical tabs -->
<Tabs {options} bind:value orientation="vertical" />

<!-- Tabs below panels -->
<Tabs {options} bind:value position="after" />

<!-- Centered tab list -->
<Tabs {options} bind:value align="center" />
```

## Editable tabs (add/remove)

```svelte
<script>
  import { Tabs } from '@rokkit/ui'

  let tabs = $state([{ value: 't1', label: 'Tab 1', content: 'Content 1' }])
  let value = $state('t1')
  let nextId = 2

  function handleAdd() {
    const id = `t${nextId++}`
    tabs = [...tabs, { value: id, label: `Tab ${nextId - 1}`, content: '' }]
    value = id
  }

  function handleRemove(v) {
    tabs = tabs.filter(t => t.value !== v)
  }
</script>

<Tabs options={tabs} bind:value editable onadd={handleAdd} onremove={handleRemove} />
```

## Tier 2 ŌĆö per-item named snippets

Set `item.snippet = 'name'` to route *specific* triggers to a named snippet; all other triggers keep the default rendering.

```svelte
<script>
  const options = [
    { value: 'overview', label: 'Overview' },
    { value: 'live',     label: 'Live',     snippet: 'live' },   // custom trigger
    { value: 'settings', label: 'Settings' }
  ]
</script>

<Tabs {options} bind:value>
  {#snippet live(proxy, selected)}
    <span class="i-lucide:radio" aria-hidden="true"></span>
    <span>{proxy.label}</span>
    <span class="size-2 rounded-full bg-danger"></span>
  {/snippet}
</Tabs>
```

The named snippet **wins over `itemContent`** ŌĆö so you can give one trigger a custom layout while leaving the rest default.

## Tier 3 ŌĆö itemContent / tabPanel / empty (blanket custom)

### `itemContent` ŌĆö replaces inner content of every trigger

The component still renders the `<button>` wrapper, `data-path`, `role="tab"`, `aria-selected`, and navigation. Use only when the default trigger renderer can't express the layout.

```svelte
<Tabs {options} bind:value>
  {#snippet itemContent(proxy, selected)}
    {#if proxy.get('icon')}
      <span class={proxy.get('icon')} aria-hidden="true"></span>
    {/if}
    <span class:font-semibold={selected}>{proxy.label}</span>
  {/snippet}
</Tabs>
```

### `tabPanel` ŌĆö replaces every panel's content

Receives a `ProxyItem`. Access panel data via `proxy.get('content')` (or any other field via `proxy.get(key)`). The component still renders the `<div data-tabs-panel>` wrapper, `role="tabpanel"`, and visibility control.

```svelte
<Tabs {options} bind:value>
  {#snippet tabPanel(proxy)}
    <div class="p-6">
      <h2 class="text-lg font-semibold">{proxy.label}</h2>
      <p>{proxy.get('content')}</p>
    </div>
  {/snippet}
</Tabs>
```

### `empty` ŌĆö rendered when `options` is empty

```svelte
<Tabs options={[]} bind:value>
  {#snippet empty()}
    <div class="p-8 text-center text-ink-mute">No tabs available</div>
  {/snippet}
</Tabs>
```

## ProxyItem API (passed to all content snippets)

| Property / method | Description |
|---|---|
| `proxy.label` | mapped display text (reads the `label` data key) |
| `proxy.value` | the raw item (for selection comparison) |
| `proxy.disabled` | whether this tab is disabled |
| `proxy.get('icon')` | mapped icon class |
| `proxy.get('content')` | panel body content (reads `content` data key) |
| `proxy.get('snippet')` | per-item snippet name |
| `proxy.get(key)` | read any field by semantic or raw name |

## Theming

### Data attributes

Outer structure:

| Attribute | Element | Notes |
|---|---|---|
| `data-tabs` | root `<div>` | also carries `data-orientation`, `data-position`, `data-align`, `data-disabled` |
| `data-tabs-list` | trigger list `<div>` | has `role="tablist"` and `aria-orientation` |
| `data-tabs-trigger` | tab trigger `<button>` | also `data-selected`, `data-disabled`, `data-path` |
| `data-tabs-panel` | panel `<div>` | has `role="tabpanel"`; also `data-panel-active` on the visible panel |
| `data-tabs-empty` | empty-state wrapper `<div>` | rendered when `options` is empty |
| `data-tabs-add` | add `<button>` | editable mode only |
| `data-tabs-remove` | remove control inside trigger | editable mode only |
| `data-tabs-placeholder` | placeholder `<div>` | shown when `value` is `undefined` |

Inner (default-rendered) content ŌĆö target these to restyle without a snippet:

| Attribute | Element |
|---|---|
| `data-tabs-icon` | icon `<span>` inside trigger |
| `data-tabs-label` | label `<span>` inside trigger |
| `data-tabs-content` | content `<div>` inside default panel |

### Styling via the `class` prop (UnoCSS arbitrary variants)

`class` lands on the root `<div data-tabs>`. Use UnoCSS arbitrary variants to reach descendants:

```svelte
<Tabs {options} bind:value class="
  [&_[data-tabs-trigger]]:px-4
  [&_[data-tabs-trigger][data-selected]]:font-semibold
  [&_[data-tabs-panel]]:p-6
" />
```

Or theme globally with named tokens in your theme CSS:

```css
[data-tabs-trigger][data-selected] { border-bottom: 2px solid var(--primary); color: var(--primary); }
[data-tabs-panel]:not([data-panel-active]) { display: none; }
```

### Horizontal tabs
`ŌåÉ`/`ŌåÆ` move focus ┬Ę `Home`/`End` ┬Ę `Enter`/`Space` select focused tab ┬Ę `Tab` moves to panel

### Vertical tabs
`Ōåæ`/`Ōåō` move focus ┬Ę `Home`/`End` ┬Ę `Enter`/`Space` select focused tab
