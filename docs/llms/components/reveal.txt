# Reveal

Scroll-triggered slide-in animation. Wraps content and animates it into view when it enters the viewport via `IntersectionObserver`.

## Import

```javascript
import { Reveal } from '@rokkit/ui'
```

## Quick Start

```svelte
<Reveal direction="up" duration={600}>
  <p>This slides up when it scrolls into view.</p>
</Reveal>

<!-- Stagger children -->
<Reveal direction="left" stagger={100}>
  <div>Card 1</div>
  <div>Card 2</div>
  <div>Card 3</div>
</Reveal>
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `direction` | `'up' \| 'down' \| 'left' \| 'right' \| 'none'` | `'up'` | Slide direction |
| `distance` | `string` | `'1.5rem'` | Slide distance (CSS value) |
| `duration` | `number` | `600` | Transition duration in ms |
| `delay` | `number` | `0` | Delay before animation starts in ms |
| `stagger` | `number` | `0` | Delay increment per direct child in ms |
| `once` | `boolean` | `true` | Only animate on first intersection |
| `threshold` | `number` | `0.1` | IntersectionObserver visibility threshold (0–1) |
| `easing` | `string` | `'cubic-bezier(0.4, 0, 0.2, 1)'` | CSS easing function |
| `class` | `string` | `''` | Additional CSS classes |

## Notes

- `stagger` applies a delay increment to each direct child element
- `once: false` re-animates every time the element scrolls in and out of view
- Uses the `reveal` action from `@rokkit/actions` internally
