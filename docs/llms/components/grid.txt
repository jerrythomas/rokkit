# Rokkit Grid Component

> Responsive selectable tile grid with keyboard navigation and selection. Built on the ProxyItem + Wrapper + Navigator stack.

Grid renders an array as a navigable CSS grid of tiles (`role="grid"`). **It is data-first: you describe items in data, and the component renders icon, label, description, badge, avatar, and shortcut automatically.** Reach for snippets only when the *markup* must differ from the default — not to add an icon or a badge (those are data).

## Usage tiers — use the SIMPLEST that works

Most grids need Tier 0 or 1. Going straight to `itemContent` is the most common mistake — it throws away the built-in rendering and you re-implement icon/label/badge by hand for every consumer.

**Tier 0 — `<Grid {items}>`** — primitives, or objects whose keys already match the defaults (`label`, `icon`, `value`, `badge`, `description`, …). No config.
```svelte
<Grid items={['Red', 'Green', 'Blue']} bind:value />

<Grid items={[
  { label: 'Rokkit',   icon: 'i-glyph:sparkle',  value: 'rokkit' },
  { label: 'Minimal',  icon: 'i-glyph:minus',     value: 'minimal' },
  { label: 'Material', icon: 'i-glyph:layers',    value: 'material', badge: 'new' }
]} bind:value />
```

**Tier 1 — `<Grid {items} {fields}>`** — same, but your data uses different key names. `fields` is a read-only remap; it never mutates your data.
```svelte
<Grid items={themes} fields={{ label: 'name', value: 'id', icon: 'symbol' }} bind:value />
```

**Tier 2 — per-item named snippets** — when *a few* tiles need bespoke markup. Set `item.snippet = 'name'`; every other tile keeps the default rendering.
```svelte
<Grid {items}>
  {#snippet featured(proxy)}…{/snippet}   <!-- only tiles with snippet:'featured' -->
</Grid>
```

**Tier 3 — `itemContent`** — blanket custom rendering for *every* tile. Last resort; you now own icon/label/badge layout yourself.

> **Before writing any snippet, ask: is this just an icon / badge / description / avatar / second line?** If yes, it's data — use Tier 0/1. Snippets are for structurally different markup (an inline control, a chart, a custom layout).

## What the default tile renders (Tiers 0–1, no snippet)

The built-in item renderer (`ItemContent`) outputs, in order, whatever fields are present:

| Field (data key) | Renders as | Notes |
|---|---|---|
| `image` (avatar) | `<img data-item-avatar>` | takes precedence over `icon` |
| `icon` | `<span data-item-icon class={icon}>` **or** `<span data-item-icon-literal>` | a CSS icon class (`i-lucide:layers`) renders as an icon; **any other string** (a single character, an emoji like `⭐`) renders as literal text. |
| `label` | `<span data-item-label>` | the display text |
| `description` | `<span data-item-description>` | secondary subtitle line, under the label |
| `badge` | `<span data-item-badge>` | trailing count / status pill |
| `shortcut` | `<kbd data-item-shortcut>` | trailing keyboard hint |

All of these are data-driven. You do **not** need a snippet to show an icon, a badge, a description, or an avatar.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `items` | `unknown[]` | `[]` | Data array (objects or primitives) |
| `fields` | `Record<string,string>` | `{}` | Remap data keys to component fields |
| `value` | `unknown` | — | Selected value — use `bind:value` |
| `size` | `'sm'\|'md'\|'lg'` | `'md'` | Tile size variant |
| `disabled` | `boolean` | `false` | Disable all tiles |
| `minSize` | `string` | `'120px'` | Minimum tile width (CSS value, e.g. `'8rem'`) |
| `gap` | `string` | `'1rem'` | Grid gap (CSS value) |
| `label` | `string` | — | Accessible label for the grid (`aria-label`) |
| `class` | `string` | `''` | Additional CSS classes on the root `<div>` |
| `onselect` | `(value, proxy) => void` | — | Called when a tile is selected |

## Field mapping (complete)

Default field names the component reads from your data. Override only the ones that differ.

| Field | Default data key | Renders |
|-------|------------------|---------|
| `label` | `'label'` | display text |
| `value` | `'value'` | selection value (falls back to the raw item) |
| `icon` | `'icon'` | icon class **or** literal char/emoji |
| `image` (avatar) | `'image'` | avatar `<img>` |
| `description` | `'description'` | secondary subtitle line |
| `badge` | `'badge'` | trailing badge/count |
| `shortcut` | `'shortcut'` | trailing `<kbd>` hint |
| `href` | `'href'` | renders tile as `<a>` (gets `aria-current="page"` when active) |
| `disabled` | `'disabled'` | disabled state |
| `snippet` | `'snippet'` | per-item named-snippet override (Tier 2) |

```svelte
<Grid {items} fields={{ label: 'name', value: 'id', description: 'summary' }} bind:value />
```

## Tier 2 — per-item named snippets

Set `item.snippet = 'name'` to route *specific* tiles to a named snippet; all other tiles keep the default rendering. Named snippets win over `itemContent`:

```svelte
<script>
  const items = [
    { label: 'Rokkit',   value: 'rokkit',   icon: 'i-glyph:sparkle' },
    { label: 'Featured', value: 'featured', icon: 'i-glyph:star', snippet: 'featured' },
    { label: 'Minimal',  value: 'minimal',  icon: 'i-glyph:minus' }
  ]
</script>

<Grid {items} bind:value>
  {#snippet featured(proxy)}
    <span class={proxy.icon} aria-hidden="true"></span>
    <span class="font-bold">{proxy.label}</span>
    <span class="text-xs text-accent">Editor's pick</span>
  {/snippet}
</Grid>
```

## Tier 3 — itemContent (blanket custom)

Replaces the inner content of **every** tile. The component still renders the `<button>` wrapper, `data-path`, and navigation. Use only when the default renderer genuinely can't express the layout.

```svelte
<Grid {items} bind:value>
  {#snippet itemContent(proxy)}
    {#if proxy.icon}<span class={proxy.icon} aria-hidden="true"></span>{/if}
    <span class="flex-1">{proxy.label}</span>
    {#if proxy.get('badge')}<span class="text-xs text-ink-mute">{proxy.get('badge')}</span>{/if}
  {/snippet}
</Grid>
```

### Interactive elements inside snippets

`stopPropagation()` so the Navigator doesn't also fire selection. Do NOT add `onclick` on an element that carries `data-path` (the tile wrapper) — the Navigator already handles it.

```svelte
{#snippet itemContent(proxy)}
  <span class="flex-1">{proxy.label}</span>
  <input type="checkbox" checked={proxy.get('checked')}
    onchange={(e) => { proxy.value.checked = e.currentTarget.checked }}
    onclick={(e) => e.stopPropagation()} />
{/snippet}
```

## ProxyItem API (passed to all content snippets)

| Property / method | Description |
|---|---|
| `proxy.label` | mapped display text |
| `proxy.icon` | mapped icon class / literal |
| `proxy.href` | mapped href |
| `proxy.value` | the original raw item (object or primitive) |
| `proxy.disabled` | whether this tile is disabled |
| `proxy.get('key')` | read any field by semantic or raw name (`proxy.get('badge')`, `proxy.get('description')`) |

## Theming

### Data attributes

Outer structure:

| Attribute | Element |
|---|---|
| `data-grid` | root `<div>` (also carries `data-size`, `data-grid-min-size`, `data-disabled`) |
| `data-grid-item` | each tile `<button>` (also `data-active`, `data-disabled`) |

The grid layout uses CSS custom properties: `--grid-min-size` (from `minSize` prop) and `--grid-gap` (from `gap` prop).

Inner (default-rendered) content — target these to restyle the built-in rendering without a snippet:

| Attribute | Element |
|---|---|
| `data-item-avatar` | avatar `<img>` |
| `data-item-icon` / `data-item-icon-literal` | icon-class span / literal-char span |
| `data-item-text` | label + description wrapper |
| `data-item-label` | label text |
| `data-item-description` | secondary line |
| `data-item-badge` | trailing badge |
| `data-item-shortcut` | trailing `<kbd>` |

### Styling via the `class` prop (UnoCSS arbitrary variants)

`class` lands on the root `<div data-grid>`. Use UnoCSS arbitrary variants to reach descendants — `&` is the root, `_` is a descendant combinator:

```svelte
<Grid {items} bind:value class="
  [&_[data-grid-item]]:rounded-lg
  [&_[data-grid-item][data-active]]:ring-2 [&_[data-grid-item][data-active]]:ring-accent
  [&_[data-item-badge]]:text-xs [&_[data-item-badge]]:text-ink-mute
" />
```

Or theme globally with named tokens in your theme CSS:

```css
[data-grid-item][data-active] { background: var(--accent-soft); color: var(--accent); }
[data-grid-item]:hover { background: var(--surface-raised); }
```

## Keyboard navigation

`←`/`→` move between tiles · `↑`/`↓` move between rows · `Home`/`End` first/last tile · `Enter`/`Space` select · `a-z` typeahead. Disabled tiles are skipped.

```javascript
import { Grid } from '@rokkit/ui'
```
