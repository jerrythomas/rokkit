# Rokkit Toggle Component

> Data-driven segmented control or cycling button for switching between a set of options. Built on the ProxyItem + Wrapper + Navigator stack.

Toggle renders options as a horizontal radio group (`variant="group"`, default) or as a single cycling button (`variant="button"`). **It is data-first: you describe options in data, and the component renders icon and label automatically.** Reach for snippets only when the *markup* must differ from the default — not to add an icon (that is data).

## Usage tiers — use the SIMPLEST that works

Most toggles need Tier 0 or 1. Going straight to `itemContent` is the most common mistake — it throws away the built-in rendering and you re-implement icon/label by hand.

**Tier 0 — `<Toggle {options}>`** — primitives, or objects whose keys already match the defaults (`label`, `value`, `icon`, …). No config.
```svelte
<Toggle options={['Daily', 'Weekly', 'Monthly']} bind:value />

<Toggle options={[
  { label: 'Day',   value: 'day',   icon: 'i-lucide:sun' },
  { label: 'Week',  value: 'week',  icon: 'i-lucide:calendar' },
  { label: 'Month', value: 'month', icon: 'i-lucide:calendar-range' }
]} bind:value />
```

**Tier 1 — `<Toggle {options} {fields}>`** — same, but your data uses different key names. `fields` is a read-only remap; it never mutates your data.
```svelte
<Toggle options={modes} fields={{ label: 'name', value: 'id' }} bind:value />
```

**Tier 2 — per-item named snippets** — when *a few* options need bespoke markup. Set `item.snippet = 'name'`; every other option keeps the default rendering.
```svelte
<Toggle {options}>
  {#snippet custom(proxy, selected)}…{/snippet}   <!-- only items with snippet:'custom' -->
</Toggle>
```

**Tier 3 — `itemContent`** — blanket custom rendering for *every* option. Last resort; you now own icon/label layout yourself.

> **Before writing any snippet, ask: is this just an icon or a label?** If yes, it's data — use Tier 0/1. Snippets are for structurally different markup (an inline indicator, a custom layout).

## What the default option renders (Tiers 0–1, no snippet)

The built-in option renderer outputs, in order, whatever fields are present:

| Field (data key) | Renders as | Notes |
|---|---|---|
| `icon` | `<span data-toggle-icon class={icon}>` | a CSS icon class (`i-lucide:sun`) rendered as a classname span |
| `label` | `<span data-toggle-label>` | the display text; also used as `aria-label` and `title` fallback |

Both are data-driven. You do **not** need a snippet to show an icon or change a label.

When `showLabels={false}`, the label span is suppressed but `label` is still used as the `title` attribute (tooltip on hover).

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'group'\|'button'` | `'group'` | `'group'` = segmented radio group; `'button'` = single cycling button |
| `options` | `unknown[]` | `[]` | Data array (strings, numbers, or objects) |
| `fields` | `Record<string,string>` | `{}` | Remap data keys to component fields |
| `value` | `unknown` | — | Selected value — use `bind:value` |
| `showLabels` | `boolean` | `true` | Show text labels alongside icons |
| `size` | `'sm'\|'md'\|'lg'` | `'md'` | Size variant |
| `disabled` | `boolean` | `false` | Disable all options |
| `label` | `string` | from messages | Accessible `aria-label` for the group container (or single button title fallback) |
| `class` | `string` | `''` | Additional CSS classes on the root element |
| `onchange` | `(value, rawItem) => void` | — | Called when selection changes |

## Field mapping (complete)

Default field names the component reads from your data. Override only the ones that differ.

| Semantic key | Default data key | Renders |
|---|---|---|
| `label` | `'label'` | display text; also `aria-label` + `title` fallback |
| `value` | `'value'` | selection value (falls back to the raw item for primitives) |
| `icon` | `'icon'` | icon class string (a CSS icon class applied to a span) |
| `disabled` | `'disabled'` | disabled state for individual options |
| `snippet` | `'snippet'` | per-item named-snippet override (Tier 2) |

```svelte
<Toggle options={modes} fields={{ label: 'name', value: 'id', icon: 'symbol' }} bind:value />
```

## Button variant

`variant="button"` renders a **single cycling button** that advances through options on each click. The button shows the *next* option (what you'd switch to). Useful for compact mode toggles where the toggle label is the action:

```svelte
<Toggle variant="button" options={[
  { value: 'system', icon: 'i-lucide:monitor', label: 'System' },
  { value: 'light',  icon: 'i-lucide:sun',     label: 'Light' },
  { value: 'dark',   icon: 'i-lucide:moon',    label: 'Dark' }
]} bind:value showLabels={false} />
```

The button variant does **not** use `role="radiogroup"` — it is a plain `<button>` that cycles. The `label` prop supplies the `aria-label` and `title` fallback.

## Tier 2 — per-item named snippets

Set `item.snippet = 'name'` to route *specific* options to a named snippet; all other options keep the default rendering.

```svelte
<script>
  const options = [
    { label: 'Edit',   value: 'edit',  icon: 'i-lucide:pen' },
    { label: 'Custom', value: 'x',     snippet: 'custom' },
    { label: 'View',   value: 'view',  icon: 'i-lucide:eye' }
  ]
</script>

<Toggle {options} bind:value>
  {#snippet custom(proxy, selected)}
    <span class="i-lucide:star"></span>
    <strong class:font-bold={selected}>{proxy.label}</strong>
  {/snippet}
</Toggle>
```

The snippet receives `(proxy: ProxyItem, selected: boolean)`.

## Tier 3 — itemContent (blanket custom)

Replaces the inner content of **every** option button. The component still renders the `<button>` wrapper, `data-path`, `role="radio"`, `aria-checked`, and navigation. Use only when the default renderer genuinely can't express the layout.

```svelte
<Toggle {options} bind:value>
  {#snippet itemContent(proxy, selected)}
    {#if proxy.get('icon')}
      <span class={proxy.get('icon')} aria-hidden="true"></span>
    {/if}
    <span class:font-semibold={selected}>{proxy.label}</span>
  {/snippet}
</Toggle>
```

## ProxyItem API (passed to all snippets)

| Property / method | Description |
|---|---|
| `proxy.label` | mapped display text (reads the `label` data key) |
| `proxy.value` | the original raw item (object or primitive) |
| `proxy.disabled` | whether this option is disabled |
| `proxy.get('icon')` | mapped icon class |
| `proxy.get('snippet')` | per-item snippet name |
| `proxy.get(key)` | read any field by semantic or raw name |

## Theming

### Data attributes

| Attribute | Element | Notes |
|---|---|---|
| `data-toggle` | root element | present on both variants |
| `data-toggle-variant` | root element | `"group"` or `"button"` |
| `data-toggle-size` | root element | `"sm"`, `"md"`, or `"lg"` |
| `data-toggle-disabled` | root element | present when all options disabled |
| `data-toggle-labels` | root element | present when `showLabels` is true |
| `data-toggle-option` | individual option `<button>` | group variant only |
| `data-selected` | option `<button>` | present on the selected option |
| `data-disabled` | option `<button>` | present on a disabled option |
| `data-path` | option `<button>` | item path key for Navigator |
| `data-toggle-icon` | icon `<span>` inside option | |
| `data-toggle-label` | label `<span>` inside option | |

### Styling via the `class` prop (UnoCSS arbitrary variants)

`class` lands on the root element. Use UnoCSS arbitrary variants to reach descendants:

```svelte
<Toggle {options} bind:value class="
  [&_[data-toggle-option]]:px-3
  [&_[data-toggle-option][data-selected]]:font-semibold
  [&_[data-toggle-label]]:text-sm
" />
```

Or theme globally with named tokens in your theme CSS:

```css
[data-toggle-option][data-selected] { background: var(--primary-soft); color: var(--primary); }
[data-toggle-option]:hover:not([data-disabled]) { background: var(--surface-raised); }
```

## Keyboard navigation (group variant)

`←`/`→` move focus · `Home`/`End` · `Enter`/`Space` select focused option · `Tab` moves focus away. Disabled options are skipped.
