# Features

Module-level features — the **what** and **why**.

## Modules

- [Data-Driven](01-Data-Driven.md) — Data adapts to components, not the other way around
- [Field Mapping](02-FieldMapping.md) — Map any data structure without transformation
- [Theming](03-Theming.md) — Theme/layout CSS separation, conforms to app styles
- [Composable Snippets](04-ComposableSnippets.md) — Extensible via snippets without modifying components
- [Accessibility](05-Accessibility.md) — Built-in keyboard navigation and ARIA support
- [State Management](06-StateManagement.md) — States package for managing component state
- [Actions](07-Actions.md) — Actions package for simplifying interactions
- [Form Generation](08-FormGeneration.md) — Schema-driven forms for variety of cases
