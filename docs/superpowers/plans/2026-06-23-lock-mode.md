# LockMode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let any subtree be pinned to a fixed color mode (`dark`/`light`) regardless of the document mode, rendering correctly across token vars, in-component `dark:` utilities, and `@rokkit/themes` component CSS.

**Architecture:** Three pieces — (1) a one-line preset change so light vars are emitted under `:root, [data-mode="light"]` (making light-locking possible); (2) a `lockMode` action that mirrors the root's `style`/`skin`/`density` onto the element via `MutationObserver` and forces `data-mode`; (3) a `LockMode` component that statically renders `data-mode` for a clean SSR paint and applies the action.

**Tech Stack:** Svelte 5 (runes, `$effect`, attachment-style actions), UnoCSS preset (`@rokkit/unocss`), Vitest, Playwright.

**Spec:** `docs/superpowers/specs/2026-06-23-lock-mode-design.md`

**Pre-flight:** verify you are on `develop` (`git branch --show-current`). All commands run from repo root. Package manager is **bun** (root scripts).

- [ ] **Step 3: Write the action**

Create `packages/actions/src/lock-mode.svelte.js`:

```js
/**
 * Lock a subtree to a fixed color mode, regardless of the document mode.
 *
 * Makes the element a theme context identical to the document root, but with
 * `mode` forced: it mirrors the root's `data-style` / `data-skin` /
 * `data-density` (the attributes `themable` writes — see themable.svelte.js)
 * onto the element and sets `data-mode` to the locked value. It follows runtime
 * style/skin/density changes via a MutationObserver on the document root; only
 * `mode` stays pinned.
 *
 * @param {HTMLElement} node
 * @param {'dark'|'light'} mode
 */

const MIRRORED = ['style', 'skin', 'density']

function syncFromRoot(node, mode) {
	const root = document.documentElement
	for (const key of MIRRORED) {
		const value = root.dataset[key]
		if (value == null) delete node.dataset[key]
		else node.dataset[key] = value
	}
	node.dataset.mode = mode
}

export function lockMode(node, mode) {
	$effect(() => {
		if (typeof document === 'undefined') return
		syncFromRoot(node, mode)
		const observer = new MutationObserver(() => syncFromRoot(node, mode))
		observer.observe(document.documentElement, {
			attributes: true,
			attributeFilter: ['data-style', 'data-skin', 'data-density']
		})
		return () => observer.disconnect()
	})
}
```

- **Types/naming consistency:** action `lockMode(node, mode)`; component `<LockMode mode>`; `LockModeProps`/`LockedMode` types; mirrored keys `['style','skin','density']` used identically in code and tests.
