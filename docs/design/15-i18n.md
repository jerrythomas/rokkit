# Internationalization

> How Rokkit externalizes its own built-in UI strings, supports RTL layouts, and
> delegates locale-aware formatting to the platform — without becoming a translation
> framework.

---

## Scope

Rokkit's i18n support covers exactly two things:

1. **Built-in component strings** — the handful of labels, placeholders, and ARIA
   annotations that components render directly (e.g. "Select an option…",
   "No results", "Loading").
2. **Layout direction** — RTL/LTR switching via `data-dir` and CSS logical properties.

Everything else — full application translation, plural rules, date locale formatting
beyond `Intl` — is out of scope. For that, use [Paraglide](https://inlang.com/m/gerre34r)
or [i18next](https://www.i18next.com/) alongside Rokkit.

---

## Messages Store

All built-in strings live in a single reactive store in `@rokkit/states`:

```javascript
// @rokkit/states/src/messages.svelte.js

const defaultMessages = {
  select: {
    placeholder: 'Select an option…',
    empty: 'No options available',
    loading: 'Loading…',
    clear: 'Clear selection'
  },
  multiselect: {
    placeholder: 'Select options…',
    empty: 'No options available',
    loading: 'Loading…',
    selected: '{n} selected'
  },
  breadcrumbs: {
    label: 'Breadcrumb navigation',
    home: 'Home'
  },
  pagination: {
    previous: 'Previous',
    next: 'Next',
    page: 'Page {n}',
    of: 'of {total}'
  },
  table: {
    empty: 'No data',
    loading: 'Loading data…',
    sortAscending: 'Sort ascending',
    sortDescending: 'Sort descending'
  },
  tree: {
    expand: 'Expand',
    collapse: 'Collapse'
  },
  datepicker: {
    placeholder: 'Select a date',
    clear: 'Clear date',
    today: 'Today'
  },
  dialog: {
    close: 'Close'
  },
  fileupload: {
    dropzone: 'Drop files here or click to upload',
    remove: 'Remove file'
  }
}

class MessagesStore {
  #messages = $state(structuredClone(defaultMessages))

  get current() {
    return this.#messages
  }

  set(overrides) {
    this.#messages = deepMerge(defaultMessages, overrides)
  }

  reset() {
    this.#messages = structuredClone(defaultMessages)
  }
}

export const messages = new MessagesStore()
```

## How Components Read Messages

Components import `messages` and read from `messages.current`:

```svelte
<script>
  import { messages } from '@rokkit/states'
</script>

<input
  placeholder={messages.current.select.placeholder}
  aria-label={messages.current.select.placeholder}
/>

{#if isEmpty}
  <div data-select-empty>{messages.current.select.empty}</div>
{/if}
```

Because `messages.current` is a `$state` object, any component reading from it will
re-render when `messages.set()` is called. Locale switching is reactive with no
additional setup.

---

## Override Pattern

Consumers override any subset of the message tree. Only the provided keys are replaced;
unspecified keys retain their defaults:

```javascript
import { messages } from '@rokkit/states'

// German overrides
messages.set({
  select: {
    placeholder: 'Option wählen…',
    empty: 'Keine Optionen verfügbar',
    loading: 'Wird geladen…'
  },
  breadcrumbs: {
    label: 'Brotkrümel-Navigation',
    home: 'Startseite'
  }
})
```

To integrate with a translation framework, call `messages.set()` whenever the active
locale changes:

```javascript
// With Paraglide
import { setLanguageTag, languageTag } from '@inlang/paraglide-js'
import { messages } from '@rokkit/states'
import * as m from './paraglide/messages.js'

function switchLocale(tag) {
  setLanguageTag(tag)
  messages.set({
    select: {
      placeholder: m.select_placeholder(),
      empty: m.select_empty()
    }
    // ...
  })
}
```

## String Key Conventions

String keys follow the pattern `componentName.slot`:

| Slot          | Meaning                                     |
| ------------- | ------------------------------------------- |
| `label`       | ARIA label for the component root           |
| `placeholder` | Input or trigger placeholder text           |
| `empty`       | Message shown when a list has no items      |
| `loading`     | Message shown during async loading          |
| `clear`       | Tooltip or SR text for a clear/reset action |
| `selected`    | Summary text when multiple items chosen     |

Tokens like `{n}` and `{total}` in string values are simple format placeholders.
Components interpolate them manually — there is no template engine:

```javascript
// Component logic
const label = messages.current.pagination.page.replace('{n}', currentPage)
```

## RTL Support

Layout direction is controlled by `data-dir` on the `<html>` element, mirroring the
`dir` HTML attribute. The `vibe` store holds the current direction and the `themable`
action writes both `dir` and `data-dir`:

```html
<html dir="rtl" data-dir="rtl"></html>
```

Using both allows HTML-native behaviour (browser text direction, form controls, Unicode
bidi algorithm) while also giving CSS the data attribute hook used by the rest of the
theming system.

### CSS logical properties

All component CSS uses logical properties. No physical direction properties appear in
theme CSS:

```css
/* Correct — direction-agnostic */
[data-input] {
  padding-inline-start: var(--density-spacing-md);
  padding-inline-end: var(--density-spacing-md);
  border-inline-start: 2px solid var(--color-primary-z5);
}

/* Wrong — do not use */
[data-input] {
  padding-left: 0.75rem;
  border-left: 2px solid var(--color-primary-z5);
}
```

### Directional icons

Icons that convey direction (chevrons, arrows, back/forward) must flip in RTL. The
approach is a CSS transform scoped to `[data-dir='rtl']`:

```css
[data-dir='rtl'] [data-icon='chevron-right'],
[data-dir='rtl'] [data-icon='chevron-left'] {
  transform: scaleX(-1);
}
```

Components set `data-icon` on icon wrapper elements. Theme CSS handles the flip.
No JavaScript direction checks appear in component logic.

## Locale-Aware Formatting

Rokkit does not implement number or date formatters. Use the platform `Intl` API
directly, or delegate to your application's formatting layer:

```javascript
// Acceptable in application code
new Intl.NumberFormat(locale, { style: 'currency', currency: 'EUR' }).format(value)
new Intl.DateTimeFormat(locale, { dateStyle: 'medium' }).format(date)
```

Rokkit components that display numbers or dates accept pre-formatted strings as props.
They do not accept raw `Date` or `number` values and format them internally.

---

## What is Not in Scope

- Plural rules and grammatical gender
- Message catalogs, `.po` files, or translation keys
- Automatic locale detection from `navigator.language`
- Right-to-left text shaping beyond CSS logical properties
- Bidirectional text mixing within a single component

Use Paraglide, i18next, or Fluent for these requirements. Rokkit's role is to expose
its own strings in a way that is easy to replace.
