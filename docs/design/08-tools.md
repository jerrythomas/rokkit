# Toolchain Design

**Status:** Complete
**Last updated:** 2026-03-27

This document covers the Rokkit toolchain: the CLI (`@rokkit/cli`), the UnoCSS preset (`@rokkit/unocss`), the icon system, and `rokkit.config.js`.

---

## Overview

The toolchain has two layers:

1. **`@rokkit/unocss` (`presetRokkit`)** — build-time UnoCSS preset that wires color tokens, icon collections, skin shortcuts, and typography into the application's CSS pipeline
2. **`@rokkit/cli`** — developer CLI for project setup, health checks, and scaffolding custom skins/themes

The central configuration file is `rokkit.config.js` in the project root. Both the preset and the CLI read from it.

---

## `rokkit.config.js`

Generated by `rokkit init`. Consumed by `presetRokkit` at build time and by CLI commands at runtime.

**Structure:**

```js
export default {
  // Color role → palette name mapping (skin)
  colors: {
    primary: 'orange',
    secondary: 'pink',
    accent: 'sky',
    surface: 'slate',
    success: 'green',
    warning: 'yellow',
    danger: 'red',
    error: 'red',
    info: 'cyan'
  },

  // Named skins (alternatives to the default color mapping)
  skins: {
    ocean: { primary: 'blue', secondary: 'teal', surface: 'zinc' }
  },

  // Icon configuration
  icons: {
    set: 'rokkit',           // which icon set to use
    path: undefined,         // path to custom icon JSON
    overrides: {}            // per-icon class overrides
  },

  // Theme variants to include
  themes: ['rokkit', 'minimal'],

  // Default theme applied on init
  defaultTheme: 'rokkit',

  // Whether to include a theme switcher component
  switcher: true,

  // Optional chart preset config
  chart: {
    colors: ['blue', 'emerald', 'rose', ...],
    shades: {
      light: { fill: '300', stroke: '700' },
      dark:  { fill: '500', stroke: '200' }
    }
  }
}
```

`loadConfig(options)` in `packages/unocss/src/config.js` reads this file and merges with defaults, so all fields are optional.

## `presetRokkit` — UnoCSS Preset

The single integration point for using Rokkit in a project:

```js
// uno.config.js
import { presetRokkit } from '@rokkit/unocss'
import config from './rokkit.config.js'

export default defineConfig({
  presets: [presetRokkit(config)]
})
```

**What `presetRokkit` registers:**

### Included presets

- `presetWind3` — Tailwind-compatible utilities, configured with data-attribute dark mode (`[data-mode="dark"]` / `[data-mode="light"]`)
- `presetTypography` — prose typography utilities
- `presetIcons` — icon class support from icon JSON collections

### Extractors

- `extractorSvelte` — scans `.svelte` files for utility classes

### Shortcuts

For each color role (primary, secondary, surface, etc.):
- z-level semantic shortcuts (`bg-surface-z2`, `text-primary-z6`, etc.) that resolve to the correct Tailwind shade pair for light/dark mode

For each named skin (from `config.skins`):
- `skin-<name>` shortcut that sets all CSS color variables for that skin

For icons:
- All `DEFAULT_ICONS` mapped to `i-rokkit-*` classes
- `config.icons.overrides` allows per-icon class substitution

### Safelist

All palette colors at every shade level are safelisted so chart and dynamic color assignments are never purged:

```
bg-blue-100, bg-blue-200, ..., bg-blue-950
bg-blue-100/50, bg-blue-200/50, ...
```

`DEFAULT_ICONS` are also safelisted.

### Preflights

CSS custom variables for the current skin are written to `:root` via a preflight, making them available at runtime before any `data-style` context.

### Font Families

Registered in the UnoCSS theme:

| Key | Fonts |
|-----|-------|
| `mono` | Victor Mono, monospace |
| `heading` | Open Sans, sans-serif |
| `sans` | Overpass, ui-serif, sans-serif |
| `body` | Open Sans, -apple-system, system-ui, ... |

## Icon System

### Curated Icon Sets

Rokkit ships a curated icon set at `@rokkit/icons/ui.json`. Icons are organized into semantic groups:

- Navigation: arrows, chevrons, close, menu, expand/collapse
- Status: check, warning, error, info, spinner
- Action: edit, delete, add, search, filter, sort
- Objects: file, folder, calendar, user, settings

### Safelisting

`DEFAULT_ICONS` in `@rokkit/core` lists all icons used by Rokkit components internally. These are added to the UnoCSS safelist, ensuring they survive tree-shaking regardless of whether they appear in scanned source files.

All other icons are tree-shaken automatically by UnoCSS's class extractor — only classes referenced in scanned `.svelte` files are included in the output.

### Custom Icon Overrides

Components reference icons via semantic shortcut names (`icon-sort-asc`, `icon-chevron-right`, etc.). These map to concrete icon classes via `iconShortcuts()`.

To override a specific icon globally:

```js
// rokkit.config.js
export default {
  icons: {
    overrides: {
      'icon-chevron-right': 'i-custom-arrow-right',
      'icon-sort-asc': 'i-custom-sort-up'
    }
  }
}
```

### Custom Icon Collections

Add a custom icon set JSON file and reference it in config:

```js
export default {
  icons: {
    collections: {
      myicons: './src/assets/icons.json'
    }
  }
}
```

The collection is passed to `presetIcons({ collections })` and becomes available as `i-myicons-*` classes.

## CLI Commands (`@rokkit/cli`)

The CLI is installed globally or run via `bunx rokkit` / `npx rokkit`.

### `rokkit init`

Interactive setup wizard. Prompts for:

1. **Color palette** — choose a preset skin (default/vibrant/seaweed) or define custom role-to-color mappings
2. **Icon set** — built-in rokkit icons or custom path
3. **Themes** — which visual themes to include (rokkit, minimal, material, frosted, etc.)
4. **Default theme** — which theme to activate on load
5. **Theme switcher** — whether to scaffold a switcher UI component
6. **Chart config** (optional) — color set (default/warm/cool) and shade contrast (standard/high/soft)

Writes:
- `rokkit.config.js` — configuration file
- `uno.config.js` — UnoCSS config importing `presetRokkit`
- Patches `app.css` with `@import '@rokkit/themes/dist/base.css'` and selected theme CSS imports
- Patches `app.html` with `data-style`, `data-mode`, `data-density` attributes on `<body>`

### `rokkit upgrade`

Upgrades all `@rokkit/*` packages to their latest published versions.

1. Reads `package.json`, extracts all `@rokkit/*` deps
2. Fetches latest version from npm for each
3. Prints a diff of current vs. latest versions
4. With `--apply`: detects the package manager (bun/pnpm/yarn/npm from lockfile) and runs the install command

### `rokkit doctor`

Health check that validates the project is correctly configured. Runs a series of checks:

| Check | What it validates |
|-------|------------------|
| `config-exists` | `rokkit.config.js` is present |
| `uno-uses-preset` | `uno.config.js` imports and calls `presetRokkit(config)` |
| `base-css-imported` | `app.css` imports `@rokkit/themes/dist/base.css` |
| `theme-css-imported` | At least one theme CSS file is imported |
| `body-has-data-style` | `app.html` `<body>` has `data-style` attribute |
| `body-has-data-mode` | `<body>` has `data-mode` attribute |
| `icons-configured` | Icon set is specified in config |

For each failing check, `doctor` prints a fix suggestion. Some checks are auto-fixable (e.g., regenerating config) — pass `--fix` to apply them.

### `rokkit skin create --name <name>`

Scaffolds a new named skin entry in `rokkit.config.js`. Adds a `skins.<name>` key with all 9 color token roles set to their default color names. The developer then edits each role to the desired palette color.

### `rokkit theme create --name <name>`

Generates a CSS theme stub file at `src/themes/<name>.css`. The stub contains one `[data-style='<name>']` block per component (25 components), each empty and ready for styling.
