**Separation of concerns:**
- **ProxyItem** вҖ” per-item reactive model with field mapping
- **ProxyTree** вҖ” collection management (create, append, addChildren, flatView)
- **Wrapper** вҖ” navigation + selection (receives ProxyTree, focus, next/prev, select, typeahead)
- **LazyWrapper** вҖ” extends Wrapper with lazy loading (onlazyload, lazy marker detection)

## Canonical Field Mapping (BASE_FIELDS)

All components share a single canonical field mapping defined in `@rokkit/core`. This replaces the scattered `DEFAULT_FIELDS`, `PROXY_ITEM_FIELDS`, and `defaultItemFields` constants.

```js
export const BASE_FIELDS = {
  // Identity
  id: 'id',                     // unique identifier (auto-generated if absent)
  value: 'value',               // selection/matching value

  // Display
  label: 'text',                // primary text (maps to 'text' for backward compat)
  icon: 'icon',                 // iconify class (e.g. 'i-lucide:home')
  avatar: 'image',              // image URL (rendered as <img>)
  subtext: 'description',       // secondary text
  tooltip: 'title',             // hover text / aria title
  badge: 'badge',               // badge content
  shortcut: 'shortcut',         // keyboard shortcut display

  // Structure
  children: 'children',         // nested items array
  type: 'type',                 // item type ('separator', 'spacer', etc.)
  snippet: 'snippet',           // named snippet key
  href: 'href',                 // navigation URL
  hrefTarget: 'target',         // link target (_blank, _self, etc.)

  // State (optional вҖ” ProxyItem manages internally when absent)
  disabled: 'disabled',
  expanded: 'expanded',
  selected: 'selected',
}
```

**Rules:**
- `icon` and `avatar` don't co-exist: `icon` renders via iconify class, `avatar` renders as `<img src>`
- Semantic keys (`label`, `subtext`, `tooltip`) map to common raw keys (`text`, `description`, `title`) for backward compatibility
- Components extend `BASE_FIELDS` only when they need component-specific fields (e.g. Tree adds `level`)
- `id` is auto-generated by ProxyItem when not present in raw data

## ProxyItem

### Class Structure

```
ProxyItem
в”ңв”Җв”Җ Constructor(raw, fields?, key?, level?)
в”Ӯ
в”ңв”Җв”Җ Private state
в”Ӯ   в”ңв”Җв”Җ #raw          вҖ” original input (never mutated by get/set)
в”Ӯ   в”ңв”Җв”Җ #item         вҖ” normalised object (= raw for objects, synthetic for primitives)
в”Ӯ   в”ңв”Җв”Җ #fields       вҖ” merged field mapping config (BASE_FIELDS + overrides)
в”Ӯ   в”ңв”Җв”Җ #id           вҖ” auto-generated unique id (from get('id') ?? generateId())
в”Ӯ   в”ңв”Җв”Җ #key          вҖ” path-based key ('0', '0-1', '0-1-2')
в”Ӯ   в”ңв”Җв”Җ #level        вҖ” nesting depth (1 = root)
в”Ӯ   в”ңв”Җв”Җ #expanded     вҖ” $state(boolean) вҖ” control state
в”Ӯ   в”ңв”Җв”Җ #selected     вҖ” $state(boolean) вҖ” control state
в”Ӯ   в””в”Җв”Җ #version      вҖ” $state(number) вҖ” bumped by set() to trigger children rebuild
в”Ӯ
в”ңв”Җв”Җ Reactive derived
в”Ӯ   в””в”Җв”Җ #children     вҖ” $derived(#buildChildren()) вҖ” auto-wrapped child ProxyItems
в”Ӯ
в”ңв”Җв”Җ Field access
в”Ӯ   в”ңв”Җв”Җ get(fieldName) вҶ’ value  вҖ” reads through field mapping
в”Ӯ   в”ңв”Җв”Җ set(fieldName, value)   вҖ” writes to #item, increments #version
в”Ӯ   в””в”Җв”Җ mutate(fieldOrObj, value?) вҖ” writes to #raw, increments #version (advanced)
в”Ӯ
в”ңв”Җв”Җ Direct getters (limited set вҖ” all others via get())
в”Ӯ   в”ңв”Җв”Җ label         вҖ” get('label') вҖ” primary display text
в”Ӯ   в”ңв”Җв”Җ value         вҖ” get('value') вҖ” selection/matching value
в”Ӯ   в”ңв”Җв”Җ id            вҖ” #id вҖ” stable unique identifier
в”Ӯ   в”ңв”Җв”Җ original      вҖ” #raw вҖ” original raw input (was `raw`)
в”Ӯ   в”ңв”Җв”Җ key, level    вҖ” structural path info
в”Ӯ   в”ңв”Җв”Җ children      вҖ” reactive child array
в”Ӯ   в”ңв”Җв”Җ hasChildren   вҖ” children.length > 0
в”Ӯ   в”ңв”Җв”Җ type          вҖ” get('type')
в”Ӯ   в””в”Җв”Җ expanded, selected вҖ” control state (get/set)
в”Ӯ
в””в”Җв”Җ Factory
    в””в”Җв”Җ _createChild(raw, fields, key, level) вҶ’ ProxyItem
        (override in subclasses for specialised children)
```

**API decisions:**
- **`label`** (not `text`): Renamed for clarity. `get('label')` resolves through field mapping to `item.text` by default.
- **`id`**: Auto-generated in constructor: `#id = get('id') ?? generateId()`. Stable across re-renders.
- **`original`** (not `raw`): Returns the original input passed to the constructor, never mutated by `get/set`.
- **`mutate(field, value)`**: Writes directly to `#raw` (the original object). Advanced operation for when the caller needs to update the source data. Also accepts an object for batch updates: `mutate({ name: 'New', age: 30 })`.
- **Limited direct getters**: Only `label`, `value`, `id` have convenience getters. All other fields accessed via `get(fieldName)` to keep the API surface small.
- **`get(fieldName)`**: Pure field mapper. Looks up `fields[fieldName]` to find the raw key, reads from `#item`. Control state (`expanded`/`selected`) accessed as direct properties, not via `get()`.
- **`set(fieldName, value)`**: Writes to `#item` through field mapping, increments `#version`.

### Control State Flow

```
                 has field in #item?
                    /          \
                  yes           no
                  /              \
         External mode      Internal mode
         read: #item[field]  read: $state
         write: #item[field] write: $state
               + $state sync
```

External mode is used when the raw data already has `expanded`/`selected` fields вҖ” the proxy reads/writes through to the raw item. Internal mode is used when the raw data lacks these fields вҖ” the proxy owns them as `$state`. Primitives always use internal mode.

### Children Rebuild Trigger

```
proxy.set('children', newArray)
  вҶ’ #item[fields.children] = newArray
  вҶ’ #version++
  вҶ’ $derived(#buildChildren()) recomputes
  вҶ’ new ProxyItem instances for new children
  вҶ’ flatView re-derives (reads proxy.children)
```

### Deprecation: ItemProxy and Proxy

**ItemProxy** (`@rokkit/ui/types/item-proxy.ts`):
- Pure TypeScript, read-only, stateless
- Used by 16 components in `@rokkit/ui`
- Features assessment:
  - Fallback resolution chains вҖ” **not needed**. Primitive handling covers stringify. Field mapping via `BASE_FIELDS` is the canonical approach.
  - `getSnippet(snippets, defaultSnippet)` вҖ” **superseded** by `resolveSnippet` utility
  - Typed `get<V>(fieldName, defaultValue)` вҖ” will add to ProxyItem when TypeScript types are added
  - `canLoadChildren` lazy marker detection вҖ” **LazyProxyItem only**. Use `loaded` property on LazyProxyItem.

**Proxy** (`@rokkit/states/proxy.svelte.js`):
- Svelte 5 reactive, depends on Ramda
- Legacy class from pre-ADR-003 architecture
- Superseded by ProxyItem: no Ramda, writable set(), path-based keys, proper expansion state

**Migration order:**
1. Migrate components from ItemProxy вҶ’ ProxyItem (16 files)
2. Remove Proxy class (also removes Ramda dependency from `@rokkit/states`)
3. Remove ItemProxy class

## ProxyTree (NEW)

### Purpose

Reactive collection manager that owns proxy instances. Provides structural mutation methods (`append`, `addChildren`) with batched version management. Derives `flatView` and `lookup` reactively.

### Class Structure

```
ProxyTree
в”ңв”Җв”Җ Constructor(items, fields?, { createProxy? })
в”Ӯ
в”ңв”Җв”Җ Private state
в”Ӯ   в”ңв”Җв”Җ #rootProxies  вҖ” $state(ProxyItem[]) вҖ” root proxy array
в”Ӯ   в”ңв”Җв”Җ #fields       вҖ” merged field mapping
в”Ӯ   в”ңв”Җв”Җ #factory      вҖ” proxy creation function
в”Ӯ   в””в”Җв”Җ #version      вҖ” $state(number) вҖ” structural version counter
в”Ӯ
в”ңв”Җв”Җ Reactive derived
в”Ӯ   в”ңв”Җв”Җ flatView      вҖ” $derived(buildReactiveFlatView(#rootProxies))
в”Ӯ   в”Ӯ                    depends on #version + proxy.expanded + proxy.children
в”Ӯ   в””в”Җв”Җ #lookup       вҖ” $derived(buildReactiveLookup(#rootProxies))
в”Ӯ                        depends on #version + proxy.children
в”Ӯ
в”ңв”Җв”Җ Mutation methods (each bumps #version exactly once)
в”Ӯ   в”ңв”Җв”Җ append(items)              вҖ” create proxies, push to #rootProxies, version++
в”Ӯ   в””в”Җв”Җ addChildren(proxy, items)  вҖ” create child proxies, proxy.set('children', ...), version++
в”Ӯ
в”ңв”Җв”Җ Read accessors
в”Ӯ   в”ңв”Җв”Җ get roots()     вҖ” root proxy array
в”Ӯ   в”ңв”Җв”Җ get flatView()  вҖ” flat view with tree line types
в”Ӯ   в””в”Җв”Җ get lookup()    вҖ” Map<key, ProxyItem>
в”Ӯ
в””в”Җв”Җ Internal
    в””в”Җв”Җ #createProxies(items, startIndex, level) вҖ” batch proxy creation
```

### Append Flow

```
proxyTree.append([item1, item2, item3])
  в”Ӯ
  в”ңв”Җв”Җ startIndex = #rootProxies.length  (e.g. 10)
  в”ңв”Җв”Җ Create proxies: key='10', key='11', key='12'
  в”ңв”Җв”Җ Push to #rootProxies array
  в””в”Җв”Җ #version++ (single bump)
       в”Ӯ
       в”ңв”Җв”Җ flatView re-derives (sees 3 new root nodes)
       в””в”Җв”Җ lookup re-derives (adds 3 new entries)
```

### AddChildren Flow

```
proxyTree.addChildren(parentProxy, [child1, child2])
  в”Ӯ
  в”ңв”Җв”Җ parentKey = parentProxy.key  (e.g. '3')
  в”ңв”Җв”Җ Create child proxies: key='3-0', key='3-1'
  в”ңв”Җв”Җ Set raw children on parent item
  в”ңв”Җв”Җ parentProxy's #version bumps internally (via set())
  в””в”Җв”Җ ProxyTree #version++ (single bump)
       в”Ӯ
       в”ңв”Җв”Җ flatView re-derives (sees children if parent expanded)
       в””в”Җв”Җ lookup re-derives (adds child entries)
```

### Version Strategy

Two version levels prevent unnecessary rebuilds:

1. **ProxyItem `#version`** вҖ” bumped by `set()`, triggers `#buildChildren()` for that proxy only
2. **ProxyTree `#version`** вҖ” bumped by `append()` and `addChildren()`, triggers `flatView` and `lookup` rebuild

For `addChildren`, both levels bump: ProxyItem version rebuilds children, ProxyTree version ensures flatView/lookup pick up the structural change.

For `append`, only ProxyTree version bumps (no existing proxy is modified).

### FlatView with Tree Lines

ProxyTree owns the `buildReactiveFlatView` function (currently in LazyWrapper). Each flatView entry:

```
{
  key: string,           // proxy.key
  proxy: ProxyItem,      // stable reference
  level: number,         // proxy.level
  hasChildren: boolean,  // children.length > 0
  isExpandable: boolean, // hasChildren || lazy marker (children === true)
  type: string,          // 'item' | 'group' | 'separator' | 'spacer'
  lineTypes: string[]    // tree line connectors for rendering
}
```

**Lazy marker pattern:** When `children` is the boolean `true` (instead of an array), it marks a node whose children haven't been fetched yet. This is detected by `LazyProxyItem.loaded === false`. When the user expands such a node, `onlazyload(item)` is called to fetch the real children. After fetch, the boolean is replaced with the actual children array.

## Wrapper

### Class Structure (after ProxyTree integration)

```

в”ңв”Җв”Җ Constructor(proxyTree, options?)
в”Ӯ   в””в”Җв”Җ Receives ProxyTree (does not create it)
в”Ӯ
в”ңв”Җв”Җ Data (delegated to ProxyTree)
в”Ӯ   в”ңв”Җв”Җ flatView      вҖ” from proxyTree.flatView
в”Ӯ   в”ңв”Җв”Җ #navigable    вҖ” $derived filter (exclude separators/spacers/disabled)

в”Ӯ   в””в”Җв”Җ collapse(path) вҖ” collapse group or move to parent
в”Ӯ
в”ңв”Җв”Җ IWrapper: selection
в”Ӯ   в”ңв”Җв”Җ select(path)   вҖ” select item or toggle group
в”Ӯ   в”ңв”Җв”Җ toggle(path)   вҖ” toggle group expansion
в”Ӯ   в”ңв”Җв”Җ extend(path)   вҖ” multi-select toggle (future)
в”Ӯ   в”ңв”Җв”Җ range(path)    вҖ” multi-select range (future)
в”Ӯ   в”ңв”Җв”Җ moveTo(path)   вҖ” sync focus to path
в”Ӯ   в”ңв”Җв”Җ moveToValue(v) вҖ” sync focus to value match
в”Ӯ   в”ңв”Җв”Җ cancel(path)   вҖ” escape (override in dropdown wrappers)
в”Ӯ   в””в”Җв”Җ blur()         вҖ” focus left (override in dropdown wrappers)
в”Ӯ
в”ңв”Җв”Җ IWrapper: typeahead
в”Ӯ   в””в”Җв”Җ findByText(query, startAfterKey?) вҶ’ key | null
в”Ӯ
в””в”Җв”Җ Callbacks
    в”ңв”Җв”Җ onselect(value, proxy)  вҖ” fired every time select() hits a leaf
    в””в”Җв”Җ onchange(value, proxy)  вҖ” fired when value differs from previous
```

**Resolved decisions:**
- **Wrapper receives ProxyTree** вҖ” does not create it. Component creates ProxyTree and passes it to Wrapper. This allows LazyWrapper to share the same ProxyTree and keeps data ownership clear.
- **No lazy loading on Wrapper** вҖ” LazyWrapper extends Wrapper for lazy-specific overrides.
- **No filter on Wrapper** вҖ” filtering is a data concern, handled before ProxyTree creation (e.g. `SearchFilter` filters items, then passes filtered items to ProxyTree).
- **Events**: `onselect(value, proxy)` fires on every selection. `onchange(value, proxy)` fires only when value differs from previous.

### Wrapper vs ProxyTree Responsibility Split

| Concern | ProxyTree | Wrapper |
|---------|-----------|---------|
| Proxy creation | Yes | No |
| flatView derivation | Yes | Reads from ProxyTree |
| lookup map | Yes | Reads from ProxyTree |
| Key generation | Yes | No |
| Append/addChildren | Yes | No |
| Focus tracking | No | Yes |
| Keyboard navigation | No | Yes (next/prev/first/last) |
| Expand/collapse | No | Yes (delegates expanded state to proxy) |
| Selection | No | Yes (onselect/onchange) |
| Typeahead | No | Yes |

```
в”Ӯ   в””в”Җв”Җ select(path)
в”Ӯ       в”ңв”Җв”Җ Group with children вҶ’ toggle expansion
в”Ӯ       в”ңв”Җв”Җ Unloaded (lazy marker) вҶ’ onlazyload(proxy.original) вҶ’ addChildren вҶ’ expand
в”Ӯ       в””в”Җв”Җ Leaf вҶ’ super.select(path) (fires onchange/onselect)
в”Ӯ
в”ңв”Җв”Җ New methods
в”Ӯ   в””в”Җв”Җ loadMore() вҶ’ onlazyload() (no args) вҶ’ proxyTree.append(result)
в”Ӯ       Called by component's "Load More" button
в”Ӯ
в””в”Җв”Җ Callbacks
    в””в”Җв”Җ onlazyload вҖ” lazy load callback (from options)
```

**Resolved decisions:**
- **Extends Wrapper** вҖ” no code duplication. Only overrides `expand()` and `select()` for lazy marker detection. All navigation (next/prev/first/last), focus management, and typeahead are inherited.
- **`toggle()` stays on Wrapper** вҖ” needed for accordion-trigger pattern. When an item has `data-accordion-trigger`, Navigator dispatches `toggle` instead of `select`. Toggle always toggles expansion regardless of current state (vs `expand` which only opens, `collapse` which only closes). Used by List groups and Tree parent nodes.
- **Since Wrapper maintains `expandedKeys` and `selectedKeys` separately**, tree refresh (via ProxyTree.append/addChildren) does not affect expansion or selection state.

### Lazy Loading Flows

**Expand unloaded node:**
```
User clicks expand on node with children: true
  вҶ’ LazyWrapper.expand(path)
  вҶ’ Detects proxy.loaded === false
  вҶ’ proxy.loading = true (spinner shows)
  вҶ’ await onlazyload(proxy.original)
  вҶ’ proxyTree.addChildren(proxy, result)
  вҶ’ proxy.expanded = true
  вҶ’ proxy.loading = false
  вҶ’ flatView re-derives (shows children)
```

**Load more root items:**
```
User clicks "Load More" button
  вҶ’ LazyWrapper.loadMore()
  вҶ’ await onlazyload() (no args)
  вҶ’ proxyTree.append(result)
  вҶ’ flatView re-derives (shows new root items)
  вҶ’ Client updates hasMore prop
```

### onlazyload Signature

```typescript
type OnLazyLoad = (current?: unknown) => Promise<unknown[]>
```

| Invocation | Meaning |
|------------|---------|
| `onlazyload(item)` | Fetch children for this node (item = raw data object) |
| `onlazyload()` | Fetch next batch of root items (pagination) |

The callback returns a Promise resolving to an array of raw items (same format as `items` prop).

### Template Structure


### Props

```svelte
let {
  items, fields, value, size, lineStyle = 'solid', icons, class: className,
  onlazyload,          // (current?) => Promise<items[]>

  onselect,
} = $props()
```

**`lineStyle` replaces `showLines`:** Instead of a boolean on/off, tree line rendering is controlled via a `data-line-style` attribute on the root element. Values: `none` | `dotted` | `dashed` | `solid` (default: `solid`). CSS themes style connectors based on this attribute. To hide lines, use `lineStyle="none"` вҖ” connectors still render for alignment but are visually hidden via CSS.
